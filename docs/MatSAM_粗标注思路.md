# MatSAM 粗标注思路说明

本文档专门说明两件事：

- `MatSAM` 到底是什么
- 它在本课题里是如何参与形成当前主监督数据的

## 1. MatSAM 是什么

`MatSAM` 可以理解为：**把通用的 `SAM` 改造成更适合材料显微图像的零样本分割方法。**

它解决的核心问题不是“训练一个新模型”，而是：

- 在没有人工标注的前提下
- 如何让 `SAM` 在材料显微图里更合理地工作

普通 `SAM` 在自然图像上很强，但在材料显微图像上会遇到两个典型问题：

- 材料组织往往是密集、分散、小尺度、重复纹理，不像自然图像那样有清晰的单个前景物体
- 如果没有合适的 prompt，`SAM` 很难知道你真正想提取的是哪一类微观结构

因此，`MatSAM` 的关键不在于重新训练，而在于：

- 先做粗 `ROI` 提取
- 自动生成更适合材料组织的 prompt
- 再结合后处理，把目标微观结构提取出来

一句话概括：

> `MatSAM = SAM + 面向材料显微图像的自动 prompt 生成 + 后处理`

## 1.1 “先做粗 ROI 提取、再自动生成 prompt”具体是什么意思

这一步如果不讲细，就很容易显得像空话。按 `MatSAM` 论文的原始思路，它主要分成两层：

### 第一层：先用传统图像方法做一个粗定位

目的不是直接把最终目标分出来，而是**先大致找到哪些地方像目标结构**。

论文中的做法是：

- 先用传统分割方法做预分割
- 典型方法包括阈值分割或边缘分割，例如 `Canny`
- 先得到一个粗糙的前景区域
- 再做一些简单清理，例如去掉异常小区域、边界平滑

这一步的结果不是最终 mask，而是一个**粗 ROI**。它告诉后面的 `SAM`：

- 哪些区域大概率有材料组织目标
- 这些目标大概分布在哪里

### 第二层：从粗 ROI 里自动生成更有针对性的 prompt

这才是 `MatSAM` 真正的关键。

论文里的逻辑是：

1. 从粗 ROI 中提取轮廓
2. 计算每个候选区域的中心点
3. 把这些中心点作为一类 prompt 点

这些点可以理解为：

- “我怀疑这里有一个材料组织目标”

但只靠这些点还不够，因为粗预分割本身会漏东西，尤其会漏：

- 低对比目标
- 形状不明显目标
- 图像边缘被裁切的不完整目标

所以 `MatSAM` 不会只用 ROI 中心点，还会再保留一类 **grid prompt**。

### 为什么还要 grid prompt

因为粗 ROI 只能保证“找到大部分容易看到的目标”，不能保证“一个不漏”。

所以 `MatSAM` 采用的是：

- `ROI 中心点`
- `grid 点`

两类 prompt 融合。

其中 `grid` 也不是原始 `SAM` 那种简单均匀铺满全图，而是做了**自适应调整**：

- 图像中部：规则分布
- 图像边缘：点更密

原因是材料显微图里边缘区域经常有：

- 被裁切一半的小目标
- 稀疏分布的小颗粒
- 不完整但仍然需要被覆盖的目标

如果边缘点和中心区域一样稀，容易直接漏掉这些结构。

### 所以整条 prompt 生成链可以写成

`粗预分割 -> ROI 轮廓 -> ROI 中心点`

再加上：

`自适应 grid 点 -> 补偿粗预分割漏掉的目标`

最后变成：

`ROI 中心点 + 自适应 grid 点 -> 输入 SAM`

### 后面 SAM 自己做什么

当 prompt 点准备好之后，后面的流程还是 `SAM` 本身的流程：

- 图像编码
- prompt 编码
- decoder 生成 mask

但 `MatSAM` 还会继续做两件事：

- 按区域裁块推理
- 再用 `NMS / soft-NMS` 去掉重复和冲突 mask

所以更完整的链条是：

`粗 ROI -> ROI 中心点 + 自适应 grid 点 -> SAM 分割 -> NMS/soft-NMS -> 最终零样本结果`

## 1.2 为什么这套思路对材料图像比原始 SAM 更合适

原始 `SAM` 默认更适合自然图像里的“少量、显著、集中”的物体。

但材料显微图像往往相反：

- 目标很多
- 分布很散
- 单个目标很小
- 整张图里都是目标，没有明显“背景主体”结构

所以如果直接用原始 `SAM` 的均匀点提示，常见问题就是：

- 中间区域还凑合
- 边缘容易漏
- 密集区域容易分不全
- 低对比目标容易直接没被注意到

`MatSAM` 的 prompt 工程，本质上就是针对这些问题做的。

## 2. 为什么本课题会先用 MatSAM

本课题一开始面临的现实问题是：

- 原始 `SEM` 图像很多
- 但高质量像素级标注几乎没有
- 如果完全靠人工从零开始逐张画，成本很高

所以前期更现实的做法不是直接训练监督模型，而是：

1. 先用 `MatSAM` 做零样本粗标注
2. 拿到一版可用但不完美的分割底稿
3. 再人工精修

也就是说，`MatSAM` 在本课题中的第一层价值，不是最终精度，而是：

- 降低初始标注成本
- 给人工精修提供起点
- 帮助尽快积累第一批高质量监督样本

## 3. 在本课题里，MatSAM 是怎么参与拿到数据的

当前仓库的真实链条是：

`原始 SEM 图像 -> MatSAM 粗标注 -> 人工精修 -> 形成 9 张高质量标注图 -> 用这 9 张图训练当前主方案`

这里要把角色分清：

- 原始数据：来自你的镍基单晶 `SEM` 图像
- `MatSAM`：负责生成粗标注底稿
- 人工精修：负责把粗标注修成可训练的高质量标签
- 当前主监督数据：就是这 `9` 张精修图

因此，现在仓库里所说的：

> “前期 `MatSAM` 粗标注后人工精修得到的 `9` 张图，是当前主要监督数据来源”

这个表述是成立的，而且逻辑链是完整的。

## 3.1 需要说明白的一点：论文思路和当前仓库实现不是完全同一层次

为了保证仓库表述严谨，这里要区分两件事：

### 论文中的 MatSAM

论文里的 `MatSAM`，重点是：

- 传统预分割拿到粗 ROI
- ROI 中心点 + 自适应 grid 点融合
- 再输入 `SAM`
- 再做 `NMS / soft-NMS`

### 当前仓库里的实际粗标注实现

当前仓库中可直接看到的脚本是：

- [`scripts/zero_shot/run_matsam.py`](../scripts/zero_shot/run_matsam.py)

这份脚本更准确地说，是一个**面向本课题实际使用的简化版零样本粗标注实现**。它保留了核心思想：

- 用 `SAM` 做零样本候选区域提取
- 结合材料图像规则做筛选
- 生成可用于人工精修的粗底稿

但它并不是论文方法的逐步、完整复刻。当前脚本里能直接看到的实际处理包括：

- 自动裁掉底部信息栏
- 使用 `SAM` 的自动 mask 生成
- 再用面积范围和亮度阈值筛掉明显不合适的候选区域

因此，仓库中最严谨的表述应该是：

> 本课题前期采用基于 `MatSAM` 思路的零样本粗标注流程，为原始 `SEM` 图像生成初始分割底稿，再通过人工精修形成 `9` 张高质量标注图。

这样写比直接说“完整复现了论文版 MatSAM”更准确。

## 4. 当前仓库里与 MatSAM 相关的两层内容

### 4.1 MatSAM 作为零样本粗标注思路

在当前仓库口径下，`MatSAM` 的主要作用是：

- 零样本对照
- 粗标注起点
- 预标注工具

而不是当前最终主模型。

### 4.2 MatSAM + SAM2 作为更实用的预标注增强版

在前期探索中，又进一步形成了：

- `MatSAM + SAM2 + localfix_strict_otsu`

这条路线的定位是：

- 比单纯 `MatSAM` 更适合做自动预标注底稿
- 更适合继续人工精修
- 但仍然不是当前论文主方案

所以现在仓库里，`MatSAM` 相关路线的正确理解是：

- `MatSAM`：零样本粗标注思路
- `MatSAM + SAM2 + localfix_strict_otsu`：更实用的自动预标注底稿路线

## 5. 为什么后来没有继续把 MatSAM 当主方案

因为本课题的后期目标已经不再只是“拿到一版粗标注”，而是要解决更难的问题：

- `SEM` 图像灰度差异小
- 边界复杂
- 小样本条件下仍要稳定分割
- 最终还要做面积分数和尺寸统计

在这些问题上，`MatSAM` 的作用是有限的：

- 它适合给出初始底稿
- 但不适合直接承担最终高精度分割与稳定统计任务

所以当 `9` 张高质量精修图形成后，主线自然就从：

`MatSAM 粗标注`

转到了：

`基于 9 张高质量标注图训练和改进监督模型`

也就是当前的 `MBU-Net++` 路线。

## 6. 当前最准确的仓库口径

如果要用一句完整的话概括，建议这样写：

> 本课题前期先采用 `MatSAM` 零样本粗标注方法，为原始 `SEM` 图像生成初始分割底稿，再通过人工精修形成 `9` 张高质量标注图，作为当前主方案训练的主要监督数据来源。在此基础上，进一步针对开题报告中提出的“小样本、低对比、复杂边界和统计稳定性”等问题，提出并验证了 `MBU-Net++` 改进方案，作为当前毕业设计的主方案。

## 7. 当前定位总结

- `MatSAM` 的价值：帮你低成本拿到粗标注起点
- `9` 张精修图的价值：提供当前最可信的监督数据
- `MBU-Net++` 的价值：解决当前课题真正的主问题

所以三者不是互相替代，而是前后衔接：

`MatSAM -> 精修标注 -> MBU-Net++`
