# 01 方案拆解

本文档把 `MBU-Net++` 方案拆成 6 个模块，并统一采用“问题 -> 设计 -> 预期收益 -> 验证方法”的结构。

## 1. 数据与标签

### 问题

当前任务的核心困难不是简单的二分类，而是以下三个叠加问题：

- `SEM` 图像灰度差异小，γ/γ′ 相界不清
- 局部亮度不均、噪声和颗粒粘连影响模型学习
- 当前只有 `9` 张精修图，样本极少，容易过拟合

### 设计

- 监督数据固定为：
  - `C:\Users\pyd111\Desktop\已完成标注图像\全部已完成标注图像_精修底图`
  - 共 `9` 组 `PNG + LabelMe JSON`
- 标签定义固定为二分类：
  - `gamma_prime = 1`
  - 其余背景 = `0`
- 首轮必须先完成 `LabelMe JSON -> 二值 mask` 批量转换
- 原图、掩膜和后续边界标签保持统一 stem，便于复现
- 首轮不设独立测试集，采用按整图划分的 `3-fold` 交叉验证
- patch 训练时严禁数据泄漏：
  - 同一原图裁出的 patch 只能出现在训练或验证中的一侧

### 预期收益

- 统一监督口径，减少后续实验歧义
- 为边界分支、`VF Loss` 和统计分析提供稳定输入
- 在超小样本条件下，优先保证结论可比较、可复现

### 验证方法

- 核查 `JSON / PNG / mask` 的 stem 是否一一对应
- 核查每个 fold 的原图名单，确认没有 patch 泄漏
- 随机抽样可视化 `mask`，确认 `gamma_prime` 填充逻辑正确

## 2. Baseline

### 问题

如果没有统一 baseline，就无法判断改进是来自结构、来自预训练，还是来自训练波动。

### 设计

首轮 baseline 只保留 3 组：

1. `U-Net`
2. `U-Net++`
3. `MicroNet-pretrained U-Net++`

统一训练配置默认值：

- 输入尺寸：默认 `256`
- 复核尺寸：`384`
- 优化器：`AdamW`
- 学习率：`1e-4`
- epoch：`80-120`
- 调度器：默认 `CosineAnnealing`
- 主损失：`Dice + BCE`
- 显存策略：默认从 `batch size = 4` 起试
- 混合精度：默认开启

### 预期收益

- 明确 `U-Net++` 相对 `U-Net` 的结构收益
- 明确 `MicroNet` 显微预训练是否带来域适配收益
- 为 `Edge Head` 和 `VF Loss` 的消融提供干净起点

### 验证方法

- 统一比较 `Dice / IoU / Precision / Recall / VF / Boundary F1`
- 每个模型使用同一组 fold 划分、同一增强版本和同一记录口径
- 报告均值、方差和代表性失败样例，而不是只保留单一最好结果

## 3. 主模型结构

### 问题

当前最难的部分是弱对比边界和颗粒粘连。单头分割模型容易学会区域占比，却不一定学会相界位置。

### 设计

主模型正式命名为 `MBU-Net++`，结构固定为：

- Encoder：`MicroNet-pretrained SE-ResNeXt50`
- Decoder：`U-Net++`
- Seg Head：输出主分割图
- Edge Head：输出边界图
- Deep Supervision：保留 `U-Net++` 多尺度输出监督

结构意图如下：

- `MicroNet` 负责显微域特征提取
- `U-Net++` 负责多尺度细节恢复和密集跳连融合
- `Edge Head` 负责把“边界在哪”单独建模
- 深监督负责稳定多尺度训练

边界标签生成固定为：

`B = dilate(Y) - erode(Y)`

首轮只比较两档边界带宽：

- `3x3`
- `5x5`

### 预期收益

- 用显微预训练增强纹理和微观形貌表征
- 用 `U-Net++` 提升细粒度结构恢复能力
- 用边界分支补强 γ/γ′ 相界识别

### 验证方法

- 与 `MicroNet-U-Net++` 做直接消融对照
- 重点看 `Boundary F1`、弱对比边界和粘连颗粒分离情况
- 保留代表性成功样例与失败样例可视化

## 4. 损失函数

### 问题

只优化区域重叠，往往会牺牲边界质量，也不保证最终面积分数统计稳定。

### 设计

总损失固定为：

`L = L_seg + λ1 L_edge + λ2 L_deep + λ3 L_vf`

其中：

- `L_seg = Dice + BCE`
- `L_edge = BCE(edge_pred, edge_gt)`
- `L_deep = 各深监督输出上的轻量 Dice + BCE`
- `L_vf = (mean(Y_hat) - mean(Y))^2`

首轮默认权重：

- `λ1 = 0.3`
- `λ2 = 0.2`
- `λ3 = 0.1`

损失设计要求固定为：

- `L_seg` 仍是主优化目标
- `L_edge` 只负责边界约束，不替代主分割
- `L_vf` 用来把训练目标与最终统计目标对齐
- 如果 `VF Loss` 带来明显边界钝化，则不保留

### 预期收益

- 主分割保持区域重叠精度
- 边界分支获得清晰监督
- `VF Loss` 约束模型不要在面积分数上偏得太远

### 验证方法

- 单独对比是否提升 `VF` 稳定性
- 同时检查 `Dice` 和 `Boundary F1`
- 不接受只提升 `VF`、却明显破坏主分割质量的结果

## 5. 小样本增强

### 问题

当前只有 `9` 张精修图。如果增强、预处理和采样策略不细化，实验波动会很大，且极易过拟合。

### 设计

首轮预处理只比较 3 组：

1. 原图
2. 原图 + `CLAHE`
3. 原图 + `CLAHE + 轻量去噪`

默认推荐把 `CLAHE` 作为首选候选，但必须通过 `E1a` 验证后才固定。

首轮增强策略固定分三层：

#### 几何增强

- 随机水平翻转
- 随机垂直翻转
- `90 / 180 / 270` 度旋转
- 小幅随机缩放

#### 灰度增强

- 轻微亮度扰动
- 轻微对比度扰动
- 轻微高斯噪声

#### 采样增强

- 在线 `patch` 采样
- 前景占比约束采样
- 边界区域优先采样

增强边界固定为：

- 不做强颜色抖动，因为当前是灰度 `SEM`
- 不做自由角度大旋转，避免插值破坏边界
- 不做强弹性形变
- 不做重模糊
- 增强强度上限以“不扭曲 γ/γ′ 颗粒真实边界”为准

patch 策略固定为两段式：

- 第一阶段：随机 `256x256` patch，作为默认主设置
- 第二阶段：只在显存允许时复核 `384x384`

patch 采样默认要求：

- 每个 epoch 混合随机 patch 和“含足够前景或边界”的 patch
- 避免大量纯背景 patch 稀释监督
- 验证阶段不做随机增强，只做确定性裁块或整图滑窗评估

监督主线稳定后，再进入两阶段半监督：

1. 先训练纯监督 `MBU-Net++`
2. 再用其对未标注图像出伪标签
3. 只保留高置信区域混入训练

高置信规则默认值：

- 前景概率 `> 0.9`：保留
- 背景概率 `< 0.1`：保留
- 中间区域：忽略

### 预期收益

- 缓解 `9` 张图带来的过拟合
- 提高小样本下的边界学习稳定性
- 为后续半监督扩样打基础

### 验证方法

- 在 `E1a` 中比较无增强与标准增强的稳定性差异
- 重点看 `VF` 和 `Boundary F1` 是否更稳
- 观察失败样例是否从“漏分边界”和“过度粘连”转为更可控误差

## 6. 系统实现

### 问题

课题最终交付不是单个模型权重，而是“分割 + 可视化 + 统计输出”的演示系统。

### 设计

系统模块边界固定为：

1. 图像输入
   - 单张
   - 批量
2. 模型推理
   - 加载最佳权重
   - 输出分割 mask
3. 结果可视化
   - 原图
   - mask
   - overlay
4. 统计输出
   - `VF`
   - 连通域数目
   - 等效直径
   - 尺寸分布
5. 导出
   - `PNG`
   - `CSV / XLSX`

系统首选演示路线固定为 `Gradio`，但明确晚于监督主线验证，不得提前占用主实验资源。

### 预期收益

- 让模型结果与统计结果可视化、可导出、可答辩展示
- 把课题从“训练脚本”推进到“系统交付”
- 与现有统计模块形成可对接接口

### 验证方法

- 以最佳监督模型为核心做单张和批量推理演示
- 确认 `VF`、连通域和尺寸分布结果可导出
- 以演示稳定性和可解释性为主，不以系统界面复杂度为优先
