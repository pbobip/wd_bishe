# 02 实验设计

本文档固定 `MBU-Net++` 研究的实验矩阵、统一训练设置、评价指标和记录要求，避免后续实验范围失控。

## 1. 当前起点数据

- 起始目录：`C:\Users\pyd111\Desktop\已完成标注图像\全部已完成标注图像_精修底图`
- 当前监督样本：`9` 张精修 `PNG`
- 配套标注：`9` 份 `LabelMe JSON`
- 首轮实验不混入其他弱标注、自动预标注或伪标签数据

当前研究假设是：

- `9` 张精修图足够支撑首轮方向验证
- 不足以支撑宽松划分或复杂模型堆叠
- 因此首轮重点不是追极限指标，而是验证增强、预训练、边界建模和统计约束是否有稳定收益

## 2. 数据划分规则

首轮数据划分固定为按整图划分的 `3-fold` 交叉验证：

- 每 fold 约 `6` 张训练、`3` 张验证
- 不建立独立测试集
- 同一原图切出的所有 patch 只能进入训练或验证的一侧
- 每 fold 的图像名单必须单独存档

约束要求：

- 不允许按 patch 随机切分数据集
- 不允许临时改 fold 只为追求更好结果
- 不允许把验证图像重新用于训练后再直接报告该轮验证指标

## 3. 统一训练设置

首轮统一训练配置默认值如下：

- 输入尺寸：`256`
- 候选复核尺寸：`384`
- 优化器：`AdamW`
- 学习率：`1e-4`
- epoch：`80-120`
- 调度器：默认 `CosineAnnealing`
- 回退调度器：`ReduceLROnPlateau`
- batch size：按 `3060 6GB` 显存自适应，默认从 `4` 起试
- 混合精度：默认开启
- 随机种子：固定
- 默认在线 `patch` 训练

训练实施要求：

- 所有 baseline 和消融实验必须共用同一组基础训练配置
- 如果因为显存不足调整输入尺寸或 batch size，必须单独记录
- 不允许在模型对比时同时更改多个核心超参数

## 4. 预处理与增强设置

这部分是首轮重点，必须单独做实验，不得含糊带过。

### 预处理候选

首轮只比较三组：

1. 原图
2. 原图 + `CLAHE`
3. 原图 + `CLAHE + 轻量去噪`

### 默认增强组合

#### 几何增强

- 水平翻转
- 垂直翻转
- `90 / 180 / 270` 度旋转
- 小幅缩放

#### 灰度增强

- 轻微亮度扰动
- 轻微对比度扰动
- 轻微高斯噪声

#### 采样增强

- 在线随机 `patch`
- 前景占比约束采样
- 边界区域优先采样

### 增强约束

- 不做强颜色增强
- 不做自由角度大旋转
- 不做强弹性形变
- 不做重模糊
- 增强强度必须服从真实颗粒边界形态

### patch 规则

- 默认训练 patch：`256x256`
- 候选复核 patch：`384x384`
- 每个 epoch 混合随机 patch 与“含前景或边界”的 patch
- 验证阶段不做随机增强，只做确定性裁块或整图滑窗评估

## 5. 统一评价指标

所有实验固定使用以下指标：

- `Dice`
- `IoU`
- `Precision`
- `Recall`
- `VF`
- `Boundary F1`

指标解释固定为：

- `Dice / IoU / Precision / Recall` 评价区域分割质量
- `VF` 评价面积分数统计偏差
- `Boundary F1` 评价相界恢复能力

结论优先级固定为：

1. 主分割质量看 `Dice / IoU`
2. 统计可靠性看 `VF`
3. 相界恢复看 `Boundary F1`
4. 是否进入系统整合，看三者平衡，而不是单一最好 Dice

## 6. 固定实验矩阵

| 实验编号 | 对比内容 | 模型或设置 | 目的 | 关键输出 |
| --- | --- | --- | --- | --- |
| `E1a` | 小样本增强对照 | `U-Net++` 无增强 vs `U-Net++` 标准增强 | 先确认增强是否必要、方向是否正确 | 各 fold 稳定性、过拟合现象、失败样例 |
| `E1` | Baseline | `U-Net` vs `U-Net++` vs `MicroNet-U-Net++` | 分离结构收益与显微预训练收益 | 基础分割指标、`VF` 对比、边界表现 |
| `E2` | 边界分支消融 | `MicroNet-U-Net++` vs `+ Edge Head` vs `+ Edge Head + Deep Supervision` | 验证边界建模是否有效 | `Boundary F1`、粘连样例可视化 |
| `E3` | `VF Loss` 消融 | `Edge` 版 vs `Edge + VF Loss` | 验证统计目标对齐是否有效 | `VF` 偏差均值、分布和副作用 |
| `E4` | 半监督增强 | 纯监督 `MBU-Net++` vs `+ 伪标签` | 仅在监督主线稳定后验证扩样收益 | 小样本下综合指标变化 |
| `E5` | 后处理影响 | 原始输出 vs 去小连通域 vs 开闭运算 + 平滑 | 只分析统计稳定性，不影响主结构结论 | `VF`、连通域数、尺寸统计变化 |

## 7. 实验顺序

实验顺序固定如下，不允许跨级并行改太多变量：

1. 先做 `E1a`
2. 在固定增强策略后做 `E1`
3. 以 `E1` 最优 baseline 为基础做 `E2`
4. 以 `E2` 最优结构为基础做 `E3`
5. 只有监督主线稳定后，才进入 `E4`
6. `E5` 只服务于统计侧分析，不参与主模型结构结论

这样做的目的，是避免结构、损失、增强和后处理同时变化，导致无法归因。

## 8. 记录模板

每次实验至少记录以下内容：

- 实验编号
- 模型结构
- encoder 初始化方式
- 是否使用 `MicroNet` 预训练
- 是否启用 `Edge Head`
- 是否启用深监督
- 是否启用 `VF Loss`
- 输入尺寸
- batch size
- epoch 数
- 优化器与调度器
- 预处理组合
- 数据增强组合
- fold 划分方式与图像名单
- 各项指标均值
- 各 fold 指标
- 代表性成功样例
- 代表性失败样例
- 显存占用和训练异常

## 9. 首轮验收标准

首轮是否继续推进，以以下标准判断：

- `E1a` 中增强后交叉验证稳定性优于或至少不差于无增强
- `MicroNet-U-Net++` 相比普通 `U-Net++`，至少在 `Boundary F1` 或 `VF` 上有明确收益
- `Edge Head` 对边界恢复有稳定提升，不只是个别 fold 偶然变好
- `VF Loss` 改善统计偏差时，不能明显破坏 `Dice`
- 如果显存或训练稳定性不足，优先收缩输入尺寸和 batch size，而不是继续堆结构
- 如果 `E2 / E3` 收益不稳定，应及时止损，把它定义为探索结果，而不是强行上升为最终主模型
