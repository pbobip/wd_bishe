# 03 实现路线

本文档把 `MBU-Net++` 方案收敛为可执行的实现顺序，避免一开始同时开太多支线。

## 1. 总原则

- 先跑通最小可验证版本，再逐步叠加结构和损失
- 先解决监督学习主线，再扩展半监督和演示系统
- 先固定数据与实验口径，再写训练代码
- 先证明收益有效，再考虑工程化整合

本轮默认数据起点固定为：

`C:\Users\pyd111\Desktop\已完成标注图像\全部已完成标注图像_精修底图`

## 2. 与现有仓库的边界

当前仓库已有：

- 展示和推理导向的后端结构
- 统计结果生成逻辑
- 旧版监督学习和迁移学习脚本参考
- 完整 `PNG` 数据池

本轮实现应采用“独立训练实验区 + 后续再择机整合”的方式推进，不直接污染现有展示平台。

默认边界如下：

- 不直接改现有后端 API
- 不直接改现有前端页面
- 不改现有阶段性结论文档
- 训练实验代码单独组织，完成验证后再考虑接回系统

## 3. 固定实施顺序

本轮实现顺序固定为：

`JSON 转 mask -> 3-fold -> 数据增强 -> MicroNet-U-Net++ baseline -> Edge Head -> VF Loss -> 半监督 -> 系统整合`

任何阶段如果前一步没有稳定通过，都不进入下一步。

## 4. 阶段 A：数据转换与数据清单

### 目标

把 `9` 组 `PNG + LabelMe JSON` 转成可复现的监督数据输入。

### 要做的事

1. 批量把 `LabelMe JSON` 转成二值 `mask`
2. 保持 `image / json / mask` 使用同一 stem
3. 生成固定的 `3-fold` 图像名单文件
4. 保存每个 fold 的训练图像和验证图像列表

### 通过标准

- 任一图像都能找到同 stem 的 `mask`
- 抽样可视化确认 `gamma_prime` 填充正确
- fold 列表固定且可复现

## 5. 阶段 B：预处理与增强基线

### 目标

先验证在 `9` 张图条件下，什么样的预处理和增强组合最稳。

### 要做的事

1. 建立 3 组预处理对照：
   - 原图
   - 原图 + `CLAHE`
   - 原图 + `CLAHE + 轻量去噪`
2. 在 `U-Net++` 上跑 `E1a`
3. 固定一组标准增强：
   - 几何增强
   - 灰度增强
   - 采样增强
4. 确认默认 patch 尺寸为 `256x256`

### 通过标准

- 增强后结果稳定性优于或至少不差于无增强
- `CLAHE` 是否保留要由实验决定，而不是先验写死
- patch 采样不会造成纯背景过多或边界监督过稀

## 6. 阶段 C：建立监督 baseline

### 目标

在统一增强和统一训练配置下完成 `E1` baseline。

### 要做的事

1. 跑通 `U-Net`
2. 跑通 `U-Net++`
3. 接入 `MicroNet-pretrained SE-ResNeXt50`
4. 跑通 `MicroNet-U-Net++`
5. 汇总 `3-fold` 指标与样例

### 通过标准

- `MicroNet` 能正常加载并训练
- 训练日志、验证指标和失败样例完整可追溯
- `MicroNet-U-Net++` 至少在边界或 `VF` 上体现研究价值

## 7. 阶段 D：加入边界分支

### 目标

验证边界建模是否能稳定改善相界恢复。

### 要做的事

1. 编写边界标签生成逻辑
2. 实现 `Edge Head`
3. 先比较 `3x3` 与 `5x5` 边界带宽
4. 以 `MicroNet-U-Net++` 为基线进行 `E2`
5. 再加入深监督版本，完成完整消融

### 通过标准

- `Edge Head` 不影响主训练流程稳定性
- `Boundary F1` 有稳定提升
- 粘连颗粒分离和弱对比边界表现更好

## 8. 阶段 E：加入 VF Loss

### 目标

让训练目标与最终统计目标对齐。

### 要做的事

1. 实现 `L_vf = (mean(Y_hat) - mean(Y))^2`
2. 固定首轮权重：
   - `λ1 = 0.3`
   - `λ2 = 0.2`
   - `λ3 = 0.1`
3. 对比 `Edge` 版与 `Edge + VF Loss`
4. 同时观察 `Dice`、`VF` 和 `Boundary F1`

### 通过标准

- `VF` 偏差更稳定
- `Dice` 没有明显下降
- 边界没有因 `VF Loss` 明显变钝

## 9. 阶段 F：半监督与后处理

### 目标

只在监督主线稳定后，再扩展样本和统计稳定性。

### 半监督默认做法

1. 用最佳纯监督 `MBU-Net++` 推理未标注原图
2. 剔除已精标图，只保留未标注池
3. 只保留高置信区域：
   - 前景概率 `> 0.9`
   - 背景概率 `< 0.1`
   - 中间区域忽略
4. 以较低样本权重混入训练

### 后处理默认做法

只在 `E5` 中比较：

- 原始输出
- 去小连通域
- 开闭运算 + 平滑

### 通过标准

- 半监督带来真实收益，而不是放大噪声
- 后处理只优化统计稳定性，不改写主结构结论

## 10. 阶段 G：系统整合

### 目标

把最佳模型以最小成本接入演示型系统。

### 要做的事

1. 加载最佳权重
2. 输出 `mask / overlay`
3. 调用统计模块输出：
   - `VF`
   - 连通域数
   - 等效直径
   - 尺寸分布
4. 支持 `PNG / CSV / XLSX` 导出
5. 优先采用 `Gradio` 演示

### 通过标准

- 单张与批量推理流程可复现
- 统计结果可导出
- 演示稳定性优先于界面复杂度

## 11. 当前默认决策

本轮实现默认做以下决策，不再反复摇摆：

- 先用 `U-Net++` 做增强对照
- baseline 固定为 `U-Net / U-Net++ / MicroNet-U-Net++`
- `Edge Head` 与深监督统一放在 `E2`
- `VF Loss` 单独放在 `E3`
- 半监督和系统整合必须晚于监督主线

如果后续进入正式代码实现，应优先建立独立实验目录，而不是把训练逻辑塞入现有推理服务代码。
