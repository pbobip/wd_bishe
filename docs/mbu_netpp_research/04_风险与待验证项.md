# 04 风险与待验证项

`MBU-Net++` 方案有明确研究价值，但不应预设它一定成为最终最优模型。本文档用于提前收敛风险、约束和止损条件。

## 1. 研究价值与现实张力

当前仓库已经有阶段性结果，因此 `MBU-Net++` 的价值在于：

- 回答“显微预训练 + `U-Net++` + 边界建模 + 统计约束”是否对当前课题难点更贴合
- 为论文中的改进模型部分提供一条逻辑完整、实验可归因的路线
- 验证它是否能在 `Boundary F1` 与 `VF` 之间取得更好的平衡

它的目标不是先验击败所有既有路线，而是先证明自己在当前课题目标下有稳定研究价值。

## 2. 主要风险表

| 风险项 | 具体表现 | 影响 | 默认应对 |
| --- | --- | --- | --- |
| 样本量极小 | 当前只有 `9` 张精修图，fold 波动可能很大 | 消融结论不稳 | 固定 `3-fold`、强化增强、统一记录失败样例 |
| `MicroNet` 接入复杂 | encoder 名称、权重键值或 `smp` 注册方式不直接兼容 | baseline 建立被拖慢 | 先单独解决 encoder 接入，再叠加其他改动 |
| 显存受限 | `3060 6GB` 可能难以稳定支撑大输入和双头结构 | 训练无法稳定复现 | 默认 `256` 输入，`384` 只做复核，不把大分辨率作为前提 |
| 边界标签带宽敏感 | `3x3` 与 `5x5` 可能导致边界过薄或过厚 | `Boundary F1` 收益不稳定 | 首轮只比较两档，不继续扩更多变体 |
| `VF Loss` 可能带来副作用 | 面积分数更稳，但边界变钝、细节流失 | 指标此消彼长 | 必须把 `VF`、`Dice`、`Boundary F1` 一起看 |
| 增强不合理 | 过强增强会破坏真实颗粒边界 | 训练学到假模式 | 严禁强颜色增强、大角度自由旋转、强弹性形变和重模糊 |
| patch 采样失衡 | 纯背景 patch 过多，或边界 patch 过少 | 监督被稀释 | 使用前景占比约束采样和边界优先采样 |
| 半监督引噪声 | 伪标签错误会放大偏差 | 小样本条件下反而退化 | 半监督明确延后，只用高置信区域，且降低权重 |

## 3. 必须验证的问题

后续推进前，至少要明确回答以下问题：

1. `MicroNet-pretrained SE-ResNeXt50` 是否能顺利接入当前训练框架
2. `MicroNet-U-Net++` 是否比普通 `U-Net++` 至少在边界或 `VF` 上更有价值
3. `Edge Head` 是否稳定提升 `Boundary F1`
4. 深监督是带来真实收益，还是只是增加训练复杂度
5. `VF Loss` 是否改善统计而不明显牺牲主分割质量
6. 小样本增强是否确实提升稳定性，而不是只让单个 fold 变好
7. 后续未标注池是否足够支撑半监督伪标签路线

## 4. 明确止损条件

出现以下情况时，应及时收缩方案，而不是继续堆复杂度：

- `MicroNet` 接入成本过高，导致 baseline 长时间无法建立
- `Edge Head` 对 `Boundary F1` 没有稳定收益
- `VF Loss` 只能通过明显牺牲 `Dice` 才带来统计改善
- 增强后的 `3-fold` 结果不稳，甚至比无增强更差
- 双头结构在当前显存条件下无法稳定训练

止损后的默认收缩路线为：

1. 先保留 `U-Net++` 主体
2. 先固定增强与预处理
3. 再决定是否继续尝试 `MicroNet`
4. 如果边界分支收益不稳，只保留 `VF` 分析，不强上双头结构

## 5. 当前默认判断

- 这条路线值得做，因为它更贴“弱对比边界 + 面积分数统计稳定性”的课题难点
- 首轮重点不是追求一次性最优，而是做出可归因、可复现的监督学习主线
- 半监督和系统整合都必须后置，不能用来掩盖监督主线没有跑稳的问题
- 如果 `E2 / E3` 收益不稳定，应在论文中如实定义为探索性改进，而不是硬推成最终主模型
