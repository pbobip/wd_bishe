- `Microstructure segmentation with deep learning encoders pre-trained on a large microscopy dataset`

- `segmentation_models.pytorch`

- 关注关键词：
  - `boundary-aware segmentation`
  - `edge supervision`
  - `deep supervision`
- 用途：
  - 为 `Edge Head` 和多尺度监督提供设计参考
  - 帮助判断边界标签带宽和监督方式是否合理

### 1.5 半监督伪标签参考

- 关注关键词：
  - `semi-supervised segmentation`

  - `confidence threshold segmentation`
- 用途：
  - 为高置信伪标签筛选策略提供参考
  - 支持后续 `E4` 的阈值与混合训练设计

## 3. 实现前待核对事项

### 3.1 `MicroNet` 接入

- `SE-ResNeXt50` 的 encoder 结构是否可直接映射到当前分割框架
- 预训练权重的 key 命名是否与本地实现一致
- 是否需要自定义 encoder 注册，而不是直接套现成权重名

### 3.2 `U-Net++` 双头封装

- `U-Net++` 是否直接扩展为双头版本，还是外包一层统一输出
- 深监督输出如何与主分割头和边界头保持训练接口一致

### 3.3 边界标签生成

- 形态学边界带是否足够稳定
- `3x3` 和 `5x5` 对低对比样本的影响是否一致
- 是否需要额外保留边界可视化脚本，便于抽查

### 3.4 统计一致性

- `VF Loss` 应按 batch 平均还是图像级计算
- `VF` 与 `Boundary F1` 是否存在明显冲突
- 后处理是否只改善统计展示，而不影响主结构结论

### 3.5 半监督扩展

- 完整 `PNG` 数据池中哪些图像尚未参与精标
- 高置信阈值是否需要按数据分布校准
- 伪标签样本与真值样本的混合比例如何控制

## 4. 建议阅读顺序

1. 先看 `MicroNet` 文献和 NASA 官方仓库，明确方案来源
2. 再看 `segmentation_models.pytorch`，明确基础实现承载体
3. 然后回看仓库内总结与历史脚本，明确当前边界和复用基础
4. 最后核对待实现事项，决定何时进入代码阶段
