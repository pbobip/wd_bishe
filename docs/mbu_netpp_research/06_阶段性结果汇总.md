# MBU-Net++ 阶段性结果汇总

更新日期：2026-03-28

## 1. 说明

当前已完成实验范围：

- `E1a`：小样本增强对照
- `E1`：`U-Net / U-Net++ / MicroNet-U-Net++` 基线对比
- `E2`：`Edge Head`、`Edge Head + Deep Supervision` 消融
- `E3`：`Edge Head + Deep Supervision + VF Loss` 消融

当前尚未完成：

- `E4`：半监督伪标签
- `E5`：后处理与统计稳定性分析
- 最终系统收口与部署版本拍板

## 2. 实验设置统一口径

- 监督金标准数据：`9` 张精修 `SEM` 图像及对应 `LabelMe JSON`
- 训练方式：按整图划分的 `3-fold`
- 每 fold：训练 `6` 张，验证 `3` 张
- patch：`256x256`
- 每 epoch 采样：`96` 个 patch
- epoch：`80`
- batch size：`2`
- 优化器：`AdamW`
- 学习率：`1e-4`
- 推理评价指标：`Dice / IoU / Precision / Recall / VF / Boundary F1`

## 3. 总结果表

| 实验 | 模型 | Dice | VF | Boundary F1 | 结论 |
| --- | --- | ---: | ---: | ---: | --- |
| `E1a` | `U-Net++` 无增强 | 0.9281 | 0.0339 | 0.7110 | 作为增强对照基线 |
| `E1a` | `U-Net++` 标准增强 | 0.9296 | 0.0425 | 0.7142 | `Dice` 和 `Boundary F1` 略升，`VF` 变差 |
| `E1` | `U-Net` | 0.9272 | 0.0368 | 0.7013 | 整体弱于 `U-Net++ / MicroNet-U-Net++` |
| `E1` | `U-Net++` | 0.9290 | 0.0302 | 0.7093 | 当前所有已完成实验中 `VF` 最低 |
| `E1` | `MicroNet-U-Net++` | 0.9305 | 0.0349 | 0.7188 | 基线中 `Dice` 和 `Boundary F1` 最好 |
| `E2` | `MicroNet-U-Net++ + Edge Head` | 0.9283 | 0.0412 | 0.7060 | 单独加 `Edge Head` 无稳定收益 |
| `E2` | `MicroNet-U-Net++ + Edge Head + Deep Supervision` | 0.9321 | 0.0371 | 0.7276 | 当前 `Dice` 和 `Boundary F1` 最优 |
| `E3` | `MicroNet-U-Net++ + Edge Head + Deep Supervision + VF Loss` | 0.9309 | 0.0332 | 0.7211 | 当前最均衡的改进方案 |

说明：`VF` 越低越好，其余指标越高越好。

## 4. 分阶段结论

### 4.1 `E1a`：数据增强的作用

- 标准增强对 `Dice` 和 `Boundary F1` 有轻微帮助。
- 这批 `9` 张精修数据上，增强没有改善 `VF`，反而让 `VF` 从 `0.0339` 变到 `0.0425`。
- 结论不是“不要增强”，而是当前增强策略更偏向提升分割泛化，而不是提升面积分数统计一致性。

### 4.2 `E1`：基线模型结论

- `U-Net` 可以作为经典 baseline，但不是最优解。
- `U-Net++` 相比 `U-Net` 更稳定，且在 `VF` 上表现最好。
- `MicroNet-U-Net++` 在 `Dice` 和 `Boundary F1` 上超过普通 `U-Net++`，说明显微预训练是有效的。

### 4.3 `E2`：边界分支是否有效

- 只加 `Edge Head` 的版本没有带来稳定收益，三项核心指标均弱于 `MicroNet-U-Net++` baseline。
- 在 `Edge Head` 基础上再加 `Deep Supervision` 后，`Dice` 和 `Boundary F1` 都提升到当前最好水平。
- 说明边界建模本身不是无效，而是需要与多尺度监督配合，单独加一个边界头不够。

### 4.4 `E3`：`VF Loss` 是否有效

- 加入 `VF Loss` 后，`VF` 从 `0.0371` 降到 `0.0332`，统计一致性明显回升。
- 同时 `Dice` 从 `0.9321` 小幅降到 `0.9309`，`Boundary F1` 从 `0.7276` 小幅降到 `0.7211`。
- 这说明 `VF Loss` 起到了“用少量分割性能换统计稳定性”的作用，但代价并不大。

## 5. 主模型建议

如果只看分割与边界恢复能力，当前最强模型是：

- `MicroNet-U-Net++ + Edge Head + Deep Supervision`

如果结合课题目标“自动分割 + 面积分数统计 + 尺寸分布统计 + 系统实现”一起看，当前更推荐作为主模型候选的是：

- `MicroNet-U-Net++ + Edge Head + Deep Supervision + VF Loss`

推荐理由：

- 它保留了较高的 `Dice`
- 它的 `Boundary F1` 仍高于 `MicroNet-U-Net++` baseline
- 它的 `VF` 明显优于 `E2` 的 `Edge + Deep` 版本
- 它更符合课题最终要输出面积分数和统计值的需求

补充说明：

- 如果论文更强调“分割精度”和“边界恢复”，可以把 `Edge + Deep` 作为最佳分割模型。
- 如果论文更强调“工程应用”和“统计可靠性”，建议把 `Edge + Deep + VF Loss` 作为最终主模型。
- `U-Net++` 的 `VF = 0.0302` 仍是当前全表最低，说明它应继续保留为强基线，不应在论文中被弱化。

## 6. 下一步

建议按以下顺序推进：

1. 固定 `Edge + Deep + VF Loss` 为当前主模型候选，整理成论文总表和实验分析。
2. 若时间允许，继续做 `E4` 半监督伪标签，验证在小样本条件下是否还能进一步提升。
3. 做 `E5` 后处理与统计稳定性实验，评估去小连通域、开闭运算和平滑对 `VF` 和尺寸统计的影响。
4. 将最终确定的模型接回推理与统计流程，收口到可演示系统。
