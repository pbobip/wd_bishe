## 2. E4：半监督伪标签

### 2.1 当前实现口径

这次 `E4` 是务实版试验，不是最初设计里的“高置信区域保留、低置信区域忽略”的完整版。

- teacher：`E2` 最优监督模型 `MicroNet-U-Net++ + Edge Head + Deep Supervision`
- 未标注池：修正裁切后的 `full_png`
- 为避免泄漏，剔除了与 `9` 张精标图重名的样本
- 实际用于伪标签的未标注图：`91` 张
- 伪标签方式：`0.5` 阈值硬伪标签
- 训练方式：每个 fold 从对应 teacher checkpoint 继续微调 `30 epochs`

### 2.2 结果

监督 baseline 来自 `E2`：

- `Dice = 0.93208`
- `IoU = 0.87350`
- `VF = 0.03709`
- `Boundary F1 = 0.72758`

半监督 `E4` 均值：

- `Dice = 0.93410`
- `IoU = 0.87688`
- `VF = 0.03916`
- `Boundary F1 = 0.73813`

相对 `E2` 的均值变化：

- `Dice +0.00202`
- `IoU +0.00337`
- `Boundary F1 +0.01054`
- `VF +0.00207`

说明：

- `Dice / IoU / Boundary F1` 是正收益
- `VF` 越小越好，所以这里是变差

### 2.3 判断

实事求是地说，`E4` 不是无效，但也不能当成成熟结论。

- 它对分割质量有小到中等幅度帮助
- 它没有改善统计目标，反而让 `VF` 误差上升
- 当前版本只能算“简化版半监督验证”

结论：

- `E4` 可以保留为有效探索
- 但不能直接作为最终主模型方案
- 如果后续继续做，应优先补“高置信保留 + 中间区域忽略”的伪标签筛选，而不是继续堆更多硬伪标签

## 3. E5：后处理

### 3.1 主口径

按最初实验设计，`E5` 比较三种输出：

- `raw`
- `remove_small`
- `morph_smooth`

这里以 `evaluate_postprocess.py` 生成的结果作为主口径，因为它最接近原计划里的“去小连通域、开闭运算和平滑”。

### 3.2 E2 底座结果

- `raw`：
  - `Dice = 0.93208`
  - `VF = 0.03709`
  - `Boundary F1 = 0.72758`
- `remove_small`：
  - `Dice = 0.93211`
  - `VF = 0.03709`
  - `Boundary F1 = 0.72832`
- `morph_smooth`：
  - `Dice = 0.89955`
  - `VF = 0.11174`
  - `Boundary F1 = 0.50541`

### 3.3 E3 底座结果

- `raw`：
  - `Dice = 0.93095`
  - `VF = 0.03323`
  - `Boundary F1 = 0.72105`
- `remove_small`：
  - `Dice = 0.93094`
  - `VF = 0.03320`
  - `Boundary F1 = 0.72182`
- `morph_smooth`：
  - `Dice = 0.89808`
  - `VF = 0.11194`
  - `Boundary F1 = 0.49403`

### 3.4 判断

`E5` 的结论很明确：

- 小连通域去除 `remove_small` 可以保留，但收益非常小，只能算微调
- 形态学平滑 `morph_smooth` 是负优化，会明显破坏分割和统计
- 后处理不是当前方案的主要增益来源

结论：

- `E5` 不能写成“后处理显著提升”
- 最多只能写成“小连通域去除可轻微改善稳定性”
- “开闭运算 + 平滑” 这条线应作为负结果保留
