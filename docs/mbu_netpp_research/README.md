# MBU-Net++ 研究区

本目录用于沉淀 `MBU-Net++` 方案的研究文档，服务于下一阶段的监督学习改进实验，不直接改动当前系统实现。

本轮固定研究主线为：

`MicroNet-pretrained SE-ResNeXt50 + U-Net++ + Edge Head + Deep Supervision + VF Loss`

## 1. 研究定位

当前仓库已经形成了阶段性结论，包括：

- 高精度边界分割路线已有既有结果
- 统计导向模型已有既有对比结论
- 零样本和预标注流程已有可用方案

本研究区的任务，不是推翻这些结论，而是围绕论文中的“针对性改进模型”部分，系统回答以下问题：

1. 在 γ/γ′ 相 `SEM` 图像场景下，显微预训练是否能比普通初始化更贴任务
2. `U-Net++` 是否能比普通 `U-Net` 更稳定地恢复弱对比边界
3. 边界建模和 `VF` 约束能否同时提升分割质量与统计可靠性
4. 在小样本前提下，如何通过增强、交叉验证和后续半监督，把方案收敛成可用模型

## 2. 固定数据起点

本轮研究的首轮监督数据固定为：

仓库内另有完整 `PNG` 原图数据池，可保留为后续伪标签半监督候选池，但不纳入首轮纯监督 baseline。

## 3. 核心研究原则

- 先把小样本监督学习主线做扎实，再谈半监督和系统演示
- 先证明结构收益和损失收益，再决定是否扩成最终主模型
- 先固定实验矩阵和记录口径，再启动代码实现
- 先做最小可验证闭环，再逐步叠加 `Edge Head` 和 `VF Loss`

本轮特别强调两点：

- 数据增强必须写细，不能停留在“做增强”这种空表述
- patch 训练必须避免数据泄漏，同一原图切出的 patch 不得同时出现在训练和验证中

## 5. 首轮研究边界

本轮研究区只负责文档化方案与后续实现规划，不直接修改：

- 现有后端 API
- 现有前端页面
- 当前展示型结果目录
- 现有阶段性结论文本

首轮明确要完成的内容是：

1. `LabelMe JSON -> 二值 mask`
2. 固定按整图划分的 `3-fold`
3. 定义并验证小样本增强策略
4. 跑通 `MicroNet-U-Net++ baseline`
5. 在 baseline 稳定后增加 `Edge Head`
6. 在结构稳定后验证 `VF Loss`

以下内容明确延后：

- 半监督伪标签
- 后处理统计扩展
- `Gradio` 演示系统
- 更复杂的大模型路线

## 6. 首轮推荐推进顺序

本轮统一推进链路固定为：

`JSON 转 mask -> 3-fold 划分 -> 预处理与增强对照 -> U-Net++ 增强基线 -> MicroNet-U-Net++ -> Edge Head -> VF Loss`

推进时必须遵守：

- 不并行改太多变量
- 不在增强、结构、损失、后处理同时变化时下结论
- 不用单次最好结果代替 `3-fold` 的整体判断
