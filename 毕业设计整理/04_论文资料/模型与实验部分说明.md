## 二、研究任务与模型目标

本毕业设计面向镍基单晶材料的 SEM 显微图像分析任务，核心问题是从灰度 SEM 图像中自动分割 `gamma_prime` 相区域，并进一步为体积分数、颗粒数量、面积、尺寸、通道宽度等统计指标提供可靠的像素级掩膜。该问题本质上属于计算机视觉中的二值语义分割任务，而不是分类、目标检测或生成任务。

模型输入为单通道灰度 SEM 图像。训练阶段的监督标签来自 LabelMe 多边形标注，数据准备脚本会将标签为 `gamma_prime` 的 polygon 转换为二值掩膜，其中前景表示目标相区域，背景表示非目标区域。模型输出为与输入图像空间尺寸对应的前景概率图，经过 sigmoid 和阈值 0.5 后得到二值分割结果。系统推理阶段还会生成 mask 图、overlay 叠加图，并将 mask 交给后端统计模块计算材料组织特征。

在整个毕设系统中，深度学习模型承担“主分割器”的作用。前端任务页面选择“深度学习”后，后端通过模型运行器调用 `MBU-Net++ 主模型`，默认权重为：

```text
D:\中期\wd_bishe\results\opt_real53_suite\experiments\opt_real53_boundary_sampling\best.pt
```

该模型在系统中的作用不是单独输出论文指标，而是为后续的图像后处理、颗粒统计、结果导出和可视化分析提供基础数据。因此模型目标不仅是提高像素重叠指标，还要尽量保证边界形态和体积分数统计稳定。

### 3.3 训练、验证、测试划分

主实验采用固定 holdout 测试集与 5 折交叉验证相结合的设计。`holdout_10_seed42.json` 将 53 张样本划分为 43 张 trainval 和 10 张固定测试样本。固定测试样本为：

```text
4233, 5, 732, 7323, 26, 2333, 20, 13, 01-14, 64
```

在 43 张 trainval 样本上构建 5 折训练验证划分，fold 0 到 fold 2 为 34 张训练、9 张验证，fold 3 到 fold 4 为 35 张训练、8 张验证。固定 holdout 的作用是保证不同模型在同一组未参与训练和验证的图像上比较，避免仅根据交叉验证结果选择模型。

### 3.4 数据预处理与增强

根据代码和配置，数据预处理包括以下步骤：

| 处理 | 代码位置 | 作用 |
| --- | --- | --- |
| LabelMe polygon 转 mask | `preparation.py::build_binary_mask` | 将人工标注转换为模型可训练的二值监督信号 |
| SEM 底栏裁切 | `preparation.py::detect_crop_height` | 去除图像底部信息栏，避免比例尺文字和底栏干扰模型 |
| 边界 mask 生成 | `preparation.py::build_edge_mask` | 为边界辅助头提供监督信号 |
| CLAHE 增强 | `backend/app/services/preprocess.py` | 提升局部对比度，增强相界区域可分性 |
| Min-Max 归一化 | `common.py::image_to_tensor` | 将不同灰度范围归一到 0 到 1，稳定训练 |
| Patch 裁剪 | `dataset.py::_random_crop` | 将较大 SEM 图像裁为 256x256 patch，降低显存压力并增加训练样本多样性 |
| 数据增强 | `dataset.py::_build_transforms` | 包括水平/垂直翻转、90 度旋转、缩放、亮度对比度扰动和高斯噪声 |
| 边界采样 | `dataset.py::_random_crop` 与配置 `min_edge_ratio` | 提高包含边界区域 patch 的出现概率，改善边界恢复 |

最终主模型 `opt_real53_boundary_sampling` 使用 CLAHE、patch size 256、minmax 归一化、随机翻转、旋转、缩放、亮度对比度扰动和高斯噪声。与第二轮 baseline 相比，主模型将 `min_edge_ratio` 从 0.002 提高到 0.005，并将 `crop_attempts` 从 12 提高到 24，因此更倾向于采到包含有效边界的训练 patch。

需要注意，裁剪和增强后的“数量”应分不同口径描述。底栏裁剪不会复制图像，因此预处理后的完整图像数量仍为 53 张，只是有效高度变为 651 到 870 像素。数据增强是在 `Dataset.__getitem__` 中在线执行，不会生成固定的扩增图片文件夹。主模型配置中每个 fold 每个 epoch 采样 192 张 256x256 增强训练图像块，训练 80 epoch，因此单个 fold 累计形成 15360 张增强训练图像块；5 折合计为 76800 张增强训练图像块。若后续按 3 个随机种子重复训练，则累计增强训练图像块为 230400 张。若将这些在线图像块离线保存为 image、mask、edge 三件套，则单次 5 折训练等价于 230400 个文件，3 seed 等价于 691200 个文件。因此论文中不应写“增强后生成 76800 张完整 SEM 原图”，而应写“5 折训练过程中累计形成 76800 张 256x256 增强训练图像块”。

按当前代码计算，数据增强不是离线复制完整图像，而是在每次读取训练样本时对 256x256 图像块进行随机增强。因此当前代码没有“增强后完整 SEM 图像数量”，只有“增强后训练图像块数量”。单次 5 折训练共有 76800 个图像块进入增强流水线；若进行 3 个随机种子重复实验，则共有 230400 个图像块进入增强流水线。按当前概率计算，单次 5 折训练中水平翻转期望触发 38400 次，垂直翻转期望触发 38400 次，`RandomRotate90` 期望调用 57600 次，其中非 0 度旋转约 43200 次；缩放仿射期望触发 23040 次，亮度/对比度扰动期望触发 19200 次，高斯噪声期望触发 15360 次。缩放、亮度/对比度和高斯噪声属于连续随机扰动，理论上可产生大量不同图像，但代码不会提前枚举或保存这些图像。

这些预处理是必要的。SEM 图像存在灰度不均、底部信息栏、标注尺寸与原图尺寸差异、前景占比较高以及边界细碎等问题。若不进行底栏裁切和局部增强，模型容易把非材料区域当作图像内容；若不进行边界采样，随机 patch 可能包含大量内部区域而缺少相界，从而影响边界分割质量。

## 四、模型结构拆解

### 4.1 整体框架

当前论文主模型可写作：

```text
MBU-Net++ with boundary-focused sampling
```

代码中对应模型名为 `mbu_netpp`，配置文件为：

```text
experiments/mbu_netpp/configs/opt_real53_boundary_sampling.yaml
```

模型主体基于 `segmentation_models_pytorch` 中的 U-Net++ 结构，编码器为 `se_resnext50_32x4d`。与普通 U-Net++ 相比，主模型引入了三个关键设计：

| 模块 | 代码实现 | 作用 |
| --- | --- | --- |
| MicroNet 显微图像预训练编码器 | `load_microscopy_pretrained_encoder` | 通过显微图像领域预训练权重初始化 encoder，提高小样本条件下的特征表达能力 |
| 边界辅助头 | `edge_head` | 在 decoder 最终特征上预测边界区域，强化相界学习 |
| 深监督分支 | `deep_heads` | 在 U-Net++ 中间 dense decoder 节点上输出辅助分割结果，缓解深层训练困难 |

代码还支持 `unet`、`deeplabv3plus`、`segformer` 和 `upernet` 等模型构建，但当前已完成并进入结果表的主线实验主要围绕 U-Net++、DeepLabV3Plus 和 MBU-Net++ 展开。SegFormer 和 UPerNet 仅在代码层支持，未在最终结果表中形成可写入论文的实验结论。

### 4.2 输入到输出的数据流

模型流程可概括为：

```text
单通道 SEM 图像
→ CLAHE 预处理与 Min-Max 归一化
→ 256x256 patch 输入
→ se_resnext50_32x4d 编码器提取多尺度特征
→ U-Net++ dense decoder 进行多尺度特征融合
→ 主分割头输出 seg_logits
→ 边界辅助头输出 edge_logits
→ 深监督头输出多个 deep_logits
→ sigmoid + threshold 得到二值 mask
→ 后端统计模块生成 Vf、颗粒数、面积、尺寸、通道宽度等指标
```

训练时，模型同时使用主分割输出、边界输出和深监督输出计算损失。推理时，系统主要使用 `seg_logits` 生成最终 mask。对于大图，`sliding_window_inference` 会使用 256x256 patch 和 0.25 overlap 滑窗推理，并对重叠区域 logits 取平均，最后裁回原始尺寸。

### 4.3 主要模块作用

编码器负责从灰度 SEM 图像中提取低层纹理、边界、孔道结构和高层组织形态特征。`se_resnext50_32x4d` 具有分组卷积和通道重标定能力，适合提取具有重复纹理和相界结构的显微图像特征。

U-Net++ 解码器通过密集跳跃连接融合浅层细节和深层语义信息。相比普通 U-Net，U-Net++ 的中间节点可以减少编码器与解码器之间的语义差距，对边界细节和多尺度组织结构更友好。

边界辅助头由 `Conv-BatchNorm-ReLU-Conv` 组成，输入为最终 decoder feature，输出 1 通道边界 logits。它不直接替代主分割结果，而是在训练阶段提供额外边界监督，使模型更关注相界位置。

深监督分支从 `x_0_1`、`x_0_2`、`x_0_3` 三个 U-Net++ dense decoder 节点引出辅助 segmentation head，并分别上采样到目标尺寸。该设计让浅层和中层 decoder 特征也接受分割监督，有助于提高梯度传播效率和中间特征的判别性。

## 五、模型原理分析

### 5.1 为什么选择该模型

本任务具有三个特点。第一，样本数量较少，主监督数据只有 53 张图像，因此模型需要利用预训练和数据增强减少过拟合风险。第二，SEM 图像中的目标相区域与背景通道之间的边界细碎且局部灰度相近，传统阈值方法难以稳定处理。第三，最终系统不仅需要 mask，还需要体积分数和颗粒形态统计，因此边界质量和前景比例误差都具有实际意义。

MBU-Net++ 适合该任务的原因在于：U-Net++ 提供多尺度特征融合能力，MicroNet 预训练提供显微图像领域的初始化，边界辅助监督提升相界识别能力，深监督改善小数据训练稳定性，边界采样进一步提高边界区域在训练 patch 中的占比。

### 5.2 损失函数

基础分割损失由 Dice Loss 和 BCEWithLogitsLoss 构成：

```text
L_seg = L_dice + L_bce
```

主模型启用边界损失和深监督损失：

```text
L_total = L_seg + 0.3 * L_edge + 0.2 * L_deep
```

其中 `L_edge` 是边界辅助头对边界 mask 的 BCE 损失，`L_deep` 是多个深监督分支分割损失的平均值。代码中还实现了 VF loss：

```text
L_vf = mean((mean(sigmoid(logits)) - mean(mask))^2)
```

但最终主模型 `opt_real53_boundary_sampling` 没有启用 VF loss。第二轮实验表明，`vf_005` 的 holdout VF Error 最低，但 Boundary F1 不如主模型，因此最终选择更均衡的边界采样模型。

### 5.3 优化器与评价指标

训练使用 AdamW 优化器，学习率为 0.0001，weight decay 为 0.0001，学习率调度为 cosine annealing，训练 80 epochs，batch size 为 6，启用 AMP 混合精度训练。

评价指标包括：

| 指标 | 含义 | 论文解释 |
| --- | --- | --- |
| Dice | 预测前景与真实前景的重叠程度 | 主要分割精度指标，越高越好 |
| IoU | 交并比 | 更严格的区域重叠指标，越高越好 |
| Precision | 预测为前景的像素中有多少是真前景 | 反映误分割前景的程度 |
| Recall | 真实前景中有多少被找回 | 反映漏分割程度 |
| VF Error | 预测前景比例与真实前景比例的绝对误差 | 反映体积分数统计偏差，越低越好 |
| Boundary F1 | 允许 2 像素容差的边界 F1 | 反映相界恢复质量，越高越好 |

这些指标可以从不同角度评价模型。Dice 和 IoU 关注整体区域一致性；Precision 和 Recall 可以判断模型偏保守还是偏扩张；VF Error 与材料统计结果直接相关；Boundary F1 则用于衡量相界边缘是否准确。

| Boundary tolerance | 2 pixels |
| Fold 数 | 5 |
| Seed | 42 |

训练流程为：准备数据 manifest，生成固定 holdout 和 5 折划分，对每个 fold 训练 80 epochs，根据验证集 Dice 保存 best.pt，汇总 5 折验证结果，选择交叉验证表现最好的 fold checkpoint 对固定 holdout 测试集进行最终评估。

### 6.3 测试流程

测试阶段使用 `holdout_eval.py`。脚本读取固定测试集 `test_stems`，加载 best checkpoint，对每张图像执行与训练一致的预处理和滑窗推理，输出预测 mask、overlay、per-image metrics 和 summary。最终主模型的 holdout 测试包含 10 张图像。

系统运行阶段使用 `backend/app/services/runners/dl_infer.py` 中的 `run_mbu_netpp`。该函数会加载 `best.pt` 中保存的 runtime config，按训练配置进行预处理、归一化、滑窗推理和阈值化，再将 mask 和 overlay 返回给后端 pipeline。后端随后计算颗粒统计和导出结果。

主模型 holdout 的逐图结果显示，大部分样本 Dice 高于 0.95。表现较弱的样本包括：

### 7.5 训练曲线与过拟合分析

主模型 5 个 fold 均训练 80 epochs，并保存验证 Dice 最优的 checkpoint。各 fold 的最佳 epoch 分别为 21、53、80、10、52。部分 fold 在较早 epoch 达到最佳，最后 epoch 的验证 Dice 略有下降，说明小样本训练中存在一定波动。由于训练代码按验证 Dice 保存 `best.pt`，最终评估使用最佳 checkpoint，可以一定程度缓解过拟合影响。

交叉验证 Dice 为 0.9606，固定 holdout Dice 为 0.9650，两者没有出现明显背离，说明主模型没有表现出严重过拟合。但 `merged78_cv5_ext10` 的 crossval Dice 达到 0.9708，而 holdout Dice 为 0.9648，提示扩展数据集的验证折与真实 holdout 的分布存在差异，论文中应避免把该实验直接当作主模型性能提升证据。

## 八、对比实验与消融实验分析

### 8.1 与传统方法对比

与最佳传统方法 Otsu+CLAHE 相比，主模型 Dice 从 0.8930 提高到 0.9650，Boundary F1 从 0.6649 提高到 0.9300，VF Error 从 0.0435 降至 0.0240。该对比说明深度模型不仅提高了区域重叠度，也显著改善了相界位置和体积分数统计。

### 8.2 与 DeepLabV3Plus 和 U-Net++ 对比

DeepLabV3Plus 是较新的语义分割模型，但在当前数据上表现不如 MBU-Net++ 主线。`real53_cv5_deeplabv3plus_imagenet` 的 holdout Dice 为 0.9595，Boundary F1 为 0.9102；第一轮最强的 `MBU + Edge + Deep` holdout Dice 为 0.9642，Boundary F1 为 0.9261。这说明当前任务中，针对显微图像和边界特征设计的 U-Net++ 改进路线比直接更换更现代的通用分割架构更有效。

与普通 U-Net++ 相比，`MBU + Edge + Deep` 的主要提升体现在边界指标。普通 U-Net++ no pretrain 的 Boundary F1 为 0.9020，而 `MBU + Edge + Deep` 为 0.9261，说明边界监督和深监督对相界恢复有实际贡献。

### 8.3 MicroNet 预训练效果

MicroNet 预训练的效果不是单调提升。U-Net++ + MicroNet 的 Boundary F1 比无预训练 U-Net++ 更高，但 holdout Dice 较低；DeepLabV3Plus + MicroNet 也没有超过 DeepLabV3Plus + ImageNet。这说明显微图像预训练可能改善边界特征，但其收益依赖模型结构和训练策略。论文中可以将 MicroNet 预训练写为“有助于增强显微纹理特征表达，但需要与边界监督和深监督结合才能稳定转化为最终性能”。

### 8.4 边界头、深监督与 VF loss

第一轮结果显示，`MBU + Edge + Deep` 优于 `MBU + Edge`，说明深监督在当前任务中有增益。第二轮提高 edge/deep 权重后，部分指标有提升，但不是所有组合都优于主模型。这说明模块本身有效，但权重过大不一定带来更好泛化。

VF loss 的作用较复杂。`opt_real53_vf_005` 将 VF Error 降至 0.0210，是第二轮中体积分数误差最低的配置，但 Boundary F1 只有 0.9190，低于主模型。因此 VF loss 更适合作为统计约束的辅助实验，不宜作为最终主模型选择依据。

### 8.5 边界采样与难样本采样

`opt_real53_boundary_sampling` 将边界采样强度提高后取得最高 Boundary F1。相比 baseline，Boundary F1 从 0.9192 提高到 0.9300，说明训练 patch 中增加边界区域比例对相界恢复有效。

`opt_real53_boundary_hard_sampling` 在边界采样基础上进一步给难样本更高采样权重，hard stems 包括 `13`、`01-14`、`5`、`26`、`64`、`32`、`36`、`3233`。该方法的 VF Error 较低，但 Boundary F1 不如纯 boundary sampling，说明难样本加权可能改善某些统计偏差，但也可能改变整体样本分布，导致泛化收益不稳定。

## 九、模型优点、局限与改进方向

### 9.1 优点

第一，模型针对 SEM 图像分割任务进行了结构和训练策略适配，而不是直接套用通用分割网络。MicroNet 预训练、边界辅助头、深监督和边界采样均与显微组织图像的边界细节相关。

第二，实验设计包含传统方法、现代深度模型、U-Net++ 对照、模块消融和外部数据扩展，能够从多个角度说明模型选择的合理性。

第三，模型已经接入毕设系统，可在前端选择“MBU-Net++ 主模型”后自动生成 mask、overlay 和统计表，实验模型与系统应用形成闭环。

第四，补充的 3 seed 重复实验表明模型稳定性较好。固定 holdout10 和 5 折划分，仅改变训练随机性时，主模型 holdout Dice 为 0.9641±0.0023，Boundary F1 为 0.9215±0.0061，VF Error 为 0.0258±0.0034。

### 9.2 局限

第一，主监督数据只有 53 张，固定测试集只有 10 张，样本规模偏小。虽然使用 5 折交叉验证缓解了数据不足问题，但最终 holdout 指标仍可能受测试样本选择影响。

第二，外部 NASA 数据与本地 SEM 图像分布并不完全一致。`merged78_cv5_ext10` 的交叉验证结果明显高于固定 real holdout，说明跨来源数据会带来分布差异，不能简单认为数据越多一定越好。

第三，当前已补充模型参数量、FLOPs、本地 CPU 推理耗时和 RTX 5090 GPU 推理耗时。GPU 结果显示，固定 holdout10 上单图平均推理耗时约 0.1152 s，含预处理总耗时约 0.1190 s。

第四，当前已完成程序化标注质量审计，能够证明图像与 JSON 配对完整、标签统一、polygon 类型正确、无无效 polygon 和无空 mask。但该审计不能替代人工标注一致性检验，不能证明不同标注人员之间的一致性，也不能完全证明边界标注精度。

### 9.3 改进方向

后续可从四个方向改进。数据方面，可扩充人工精标样本，并建立双人标注或抽样复核机制。模型方面，可引入轻量级边界细化模块或多尺度注意力，但应避免过度堆叠模块。训练方面，可尝试 threshold 校准、边界损失权重动态调整、困难样本 curriculum 训练。评价方面，应增加 GPU 推理速度、外部测试集和多随机种子实验。

## 十、论文写作化整理

以下内容可直接作为“模型设计与实验分析”章节初稿使用。表格中的具体数值来自当前仓库结果文件。

### 10.1 模型任务描述

本文研究对象为镍基单晶材料 SEM 显微图像中的 `gamma_prime` 相区域分割。该任务属于二值语义分割问题，模型输入为单通道灰度 SEM 图像，输出为与输入图像尺寸对应的二值分割掩膜。分割结果进一步用于计算体积分数、颗粒数量、面积、尺寸以及通道宽度等材料组织统计指标。与普通图像分类任务不同，本任务要求模型在像素级别准确区分目标相与背景区域，尤其需要保持相界边缘的连续性和准确性。

在系统实现中，深度学习模型作为自动分割模块嵌入到 SEM 分割统计平台。用户导入图像后，系统调用 MBU-Net++ 主模型生成 mask 和叠加图，再由统计模块完成材料组织参数计算和结果导出。因此，模型性能直接影响后续统计分析的可靠性。

### 10.2 数据集与预处理

本文主实验使用 53 张 SEM 图像及其 LabelMe 标注文件构建监督数据集。所有标注对象均为 `gamma_prime` 标签，共包含 4268 个多边形标注。数据准备阶段将多边形标注转换为二值掩膜，并自动裁切 SEM 图像底部信息栏，以减少比例尺和文字信息对模型训练的干扰。同时，根据二值掩膜生成边界监督图，用于训练边界辅助分支。

程序化审计结果显示，53 张原图和 53 个 JSON 标注文件全部配对成功，所有标注 shape 均为 polygon，标签均为 `gamma_prime`，无无效 polygon 和空 mask。裁剪后完整图像数量仍为 53 张，平均有效高度约 677.5 像素，平均宽度约 1019.2 像素。审计中发现 148 个 polygon 点超出图像边界，数据准备脚本会对其进行裁剪，论文中可将其作为标注边界警告说明。

标注规范可由 `D:\QQ\3325314121标注指南.txt` 支撑。该指南明确标注对象为镍基单晶 γ′ 强化相，要求使用 LabelMe 或 EISeg 等支持 polygon 的标注工具，禁止使用矩形框，禁止将多个颗粒圈为一个整体，并强调颗粒之间的 γ 通道需要保留。对于筏化结构，若颗粒明显连体则标为一个对象，若边界不清晰则遵循“宁缺毋滥”的原则。该规范可作为论文中“标注原则与标签质量控制”的依据。

SEM 图像的部分采集与标定信息可以从原图底部信息栏获取，例如比例尺、放大倍数、FOV、WD 等。系统代码中也包含 SEM ROI 检测与 OCR 标定入口，可用于辅助读取底部信息栏。但若底部信息栏中没有材料牌号、设备型号或加速电压等字段，则这些信息仍需要从实验记录或图像来源说明中人工补充。

数据集采用固定测试集与 5 折交叉验证结合的划分方式。首先从 53 张图像中固定选取 10 张作为 holdout 测试集，其余 43 张用于 5 折训练验证。该设计既能利用有限数据进行交叉验证，又能在完全独立的固定测试集上比较不同模型。

训练时，图像经过 CLAHE 局部对比度增强和 Min-Max 归一化，并随机裁剪为 256x256 图像块。数据增强包括水平翻转、垂直翻转、90 度旋转、缩放、亮度对比度扰动和高斯噪声。增强方式为在线增强，不生成固定扩增数据集。主模型每个 fold 每个 epoch 采样 192 张增强训练图像块，训练 80 epoch，因此单个 fold 约包含 15360 张增强训练图像块，5 折训练过程合计约 76800 张增强训练图像块。最终主模型还使用边界采样策略，提高包含相界区域图像块的采样概率，从而增强模型对边界细节的学习。

### 10.3 模型结构设计

本文最终采用的模型为 MBU-Net++ with boundary-focused sampling。该模型以 U-Net++ 为基础框架，编码器采用 `se_resnext50_32x4d`，并使用显微图像领域的 MicroNet 预训练权重初始化。U-Net++ 的密集跳跃连接能够融合浅层纹理细节与深层语义信息，适合处理 SEM 图像中多尺度组织结构和复杂边界。

在基础 U-Net++ 结构上，本文加入边界辅助头和深监督分支。边界辅助头在最终解码特征上预测相界区域，使模型在训练阶段显式关注目标相边界。深监督分支从多个中间解码节点输出辅助分割结果，改善梯度传播并提高中间特征的判别性。模型训练阶段同时优化主分割输出、边界输出和深监督输出；推理阶段使用主分割输出生成最终二值 mask。

### 10.4 模型训练过程

模型训练使用 AdamW 优化器，学习率为 0.0001，weight decay 为 0.0001，训练 80 epochs，batch size 为 6，并使用 cosine annealing 学习率调度和 AMP 混合精度训练。损失函数以 Dice Loss 和 BCEWithLogitsLoss 为基础，同时加入边界辅助损失和深监督损失。最终主模型的总损失可表示为：

```text
L_total = L_seg + 0.3 * L_edge + 0.2 * L_deep
```

其中 `L_seg` 为主分割损失，`L_edge` 为边界预测损失，`L_deep` 为深监督分支损失。训练过程中，每个 fold 根据验证集 Dice 保存最佳 checkpoint，最终对固定 holdout 测试集进行评估。

### 10.5 实验环境与参数设置

第二轮完整实验在 NVIDIA GeForce RTX 5090 GPU 上完成，显存为 31.36 GB。软件环境包括 Python 3.12.3、PyTorch 2.8.0+cu128、CUDA、OpenCV、Albumentations 和 segmentation_models_pytorch。主模型输入 patch size 为 256，滑窗推理 overlap 为 0.25，二值化阈值为 0.5，Boundary F1 计算时使用 2 像素容差。

主模型参数量为 50,986,709。以 256x256 单 patch 为输入时，仅统计 Conv2d 和 Linear 的计算量为 57.6856 GMACs，按 multiply-add=2 口径折算为 115.3712 GFLOPs。本地 CPU 环境下，单 patch 前向平均耗时约 0.3593 s；固定测试集单图平均需要 20 个滑窗 patch，CPU 单图推理耗时约 7.6539 s。RTX 5090 环境下，单 patch 前向平均耗时约 0.0065 s，固定测试集单图平均推理耗时约 0.1152 s，含预处理总耗时约 0.1190 s。

### 10.6 实验结果分析

传统方法在固定测试集上的最佳 Dice 约为 0.893，明显低于深度学习模型。最终主模型在固定 holdout 测试集上取得 Dice 0.964984、IoU 0.932976、Precision 0.954685、Recall 0.976702、VF Error 0.024045 和 Boundary F1 0.929954。结果表明，模型能够较准确地恢复目标相区域，并在边界质量和体积分数误差方面优于传统图像处理方法。

从 Precision 和 Recall 看，主模型 Recall 高于 Precision，说明模型整体更倾向于减少漏分割，部分样本可能存在轻微前景扩张。VF Error 约为 0.024，表示预测体积分数与真实标注体积分数的平均绝对误差较小，可以支撑后续统计分析。逐图结果中，样本 `13` 和 `01-14` 的边界指标相对较低，说明模型在纹理复杂或边界模糊图像上仍存在改进空间。

为了支撑论文中的可视化结果分析，本文从固定 holdout10 测试集中选取三类代表样本：`13` 作为困难样本，其 MBU-Net++ Dice 为 0.916；`5` 作为接近平均水平的中等样本，其 Dice 为 0.963；`2333` 作为容易样本，其 Dice 为 0.982。已生成传统方法 Otsu+CLAHE 与 MBU-Net++ 的可视化对比图，路径为 `results/paper_figures/holdout_traditional_vs_mbu_typical_comparison.png`。同时生成传统方法与深度学习方法的指标柱状对比图，路径为 `results/paper_figures/holdout_traditional_vs_mbu_metrics_bar.png`。这两张图可分别用于论文“定性结果分析”和“传统方法 baseline 对比”部分。

### 10.7 对比实验与消融实验

模型路线对比表明，DeepLabV3Plus 和普通 U-Net++ 均未超过 MBU-Net++ 主线。第一轮实验中，`MBU + Edge + Deep` 的 holdout Dice 为 0.9642，Boundary F1 为 0.9261，优于 DeepLabV3Plus+ImageNet 和普通 U-Net++。这说明针对边界信息和显微图像特征进行结构适配，比单纯更换通用分割架构更有效。

第二轮消融实验显示，边界采样是最终模型性能提升的关键因素。`opt_real53_boundary_sampling` 相比第二轮 baseline 将 holdout Dice 从 0.9633 提高到 0.9650，将 Boundary F1 从 0.9192 提高到 0.9300。VF loss 可以降低体积分数误差，但会牺牲部分边界指标，因此未作为最终主模型。NASA train-only 实验在 Dice 上略高，但边界指标略低，适合作为外部相似数据辅助训练的补充实验。

### 10.8 模型优势与不足

本文模型的优势在于：结合了显微图像预训练、多尺度特征融合、边界辅助监督、深监督和边界采样策略，能够在小样本 SEM 数据集上取得较高分割精度，并已接入实际系统完成 mask、overlay 和统计结果输出。实验设计包含传统方法、深度学习基线、模块消融和外部数据扩展，能够较完整地支撑模型有效性。

不足在于：主数据集规模较小，固定测试集只有 10 张；外部 NASA 数据与本地 SEM 图像存在分布差异；标注质量已有程序化审计，但仍缺少双人标注一致性或人工抽样复核记录。后续可从扩大精标数据、增加外部测试集和优化困难样本训练策略等方向继续改进。

### 10.9 本章小结

本章围绕 SEM 图像中 `gamma_prime` 相的自动分割任务，设计并验证了基于 U-Net++ 的 MBU-Net++ 模型。模型通过 MicroNet 显微图像预训练、边界辅助监督、深监督和边界采样策略提高了小样本条件下的分割性能。实验结果表明，本文方法在固定 holdout 测试集上显著优于传统阈值和边缘方法，并优于 DeepLabV3Plus 与普通 U-Net++ 等对照模型。最终主模型取得 Dice 0.9650 和 Boundary F1 0.9300，能够为后续材料组织统计提供较可靠的分割基础。

### 12.1 系统定位

本毕业设计系统面向扫描电镜图像中的镍基单晶微观组织分析，目标是将 SEM 图像导入、图像预处理、γ/γ′ 相分割、结果可视化、统计分析和报告导出整合为一个 Web 平台。系统前端提供任务创建、图像导入、模型选择、运行监控、结果查看和统计图表展示；后端负责图像处理流程编排、传统算法分割、深度学习模型推理、统计指标计算以及结果文件导出。

在整个毕业设计中，深度学习模型负责将 SEM 原图转换为 γ′ 相二值 mask，是后续面积、数量、粒径、体积分数和通道宽度等统计指标的基础。系统平台则负责把模型能力封装为可操作的软件流程，使用户能够批量处理图像并获得可视化和结构化结果。

### 12.2 系统总体架构

系统采用前后端分离架构。前端基于 Vue 3、Vite、TypeScript、Element Plus、Pinia 和 ECharts 实现，主要承担用户交互、任务配置、结果展示和统计图表渲染。后端基于 FastAPI、SQLite、SQLAlchemy、OpenCV、scikit-image、NumPy、Matplotlib、RapidOCR、PyTorch 子进程和 python-docx 实现，承担数据存储、流程调度、图像分割、统计计算和报告生成。

系统后端的核心流程由 `backend/app/services/pipeline.py` 进行编排，主要包括输入解析、预处理、分割、统计、导出和任务状态更新。传统分割算法位于 `backend/app/services/algorithms/traditional.py`，深度学习推理由 `backend/app/services/model_runner.py` 调用 `backend/app/services/runners/dl_infer.py` 子进程完成。统计分析由 `backend/app/services/statistics.py` 实现，导出服务由 `backend/app/services/exporter.py` 与 `backend/app/services/report_service.py` 实现。

系统实现的主要代码依据如下。

### 12.3 系统功能流程

系统运行时，用户首先在前端 `/tasks/create` 页面创建分析任务并上传 SEM 图像。前端可调用 `/api/calibration/inspect` 或 `/api/calibration-hint` 对 SEM 图像底部信息栏、比例尺和标定信息进行探测，随后通过 `/api/runs` 创建任务，通过 `/api/runs/{run_id}/images` 上传图像，通过 `/api/runs/{run_id}` 更新任务配置，并通过 `/api/runs/{run_id}/execute` 提交执行。任务提交后，后端将图像保存到存储目录，并建立任务、图像、运行步骤和结果记录。随后流程引擎依次执行 `input → preprocess → traditional 或 dl → stats → export`。

对于深度学习分割，系统通过模型运行器生成子进程配置文件，并调用独立推理脚本。推理脚本读取输入图像、模型槽位、权重路径、输入尺寸、运行设备和阈值等参数，输出预测 mask、overlay 图像、进度文件和 manifest 结果文件。主流程再读取推理产物并进入统计与导出阶段。

统计模块根据 mask 与图像标定信息计算 γ′ 相面积分数、颗粒数量、单个颗粒面积、等效直径、尺寸分布、通道宽度等指标。任务完成后，前端通过 `/api/runs/{run_id}` 和 `/api/runs/{run_id}/results` 获取任务状态、图像结果、步骤记录、指标记录和导出文件。后处理预览与确认接口为 `/api/runs/{run_id}/postprocess/preview` 和 `/api/runs/{run_id}/postprocess/confirm`，统计详情页面路由为 `/runs/:id/statistics`。导出模块支持生成结构化 JSON、XLSX、ZIP 结果包和 Word 报告，为论文实验记录和答辩演示提供材料。

### 12.4 系统侧可写入论文的客观表述

本文设计并实现了一个面向 SEM 图像微观组织分析的 Web 系统。系统采用 Vue 3 与 FastAPI 构建前后端分离架构，前端负责图像导入、参数配置、运行状态展示和统计结果可视化，后端负责图像预处理、分割推理、统计计算和结果导出。系统将传统图像处理方法与深度学习分割模型统一封装为可选的分割运行器，使用户能够在同一任务流程中完成不同算法路线的对比分析。

深度学习推理模块采用独立子进程方式运行，主服务通过配置文件向推理脚本传递输入路径、模型槽位、权重路径、输入尺寸、设备和阈值等参数。该设计降低了 Web 服务与深度学习环境之间的耦合度，并便于后续扩展不同模型权重。模型推理结果以 mask 和 overlay 图像形式保存，随后由统计模块计算面积分数、颗粒数量和尺寸分布等指标，最终通过导出模块生成表格、图像和报告文件。

系统实现层面已覆盖从 SEM 图像输入到分割结果、统计结果和报告导出的完整流程。当前系统更适合作为毕业设计中的实验验证与辅助分析平台；若用于生产环境，还需要进一步完善异常提示、模型权重管理、任务取消、权限控制和更严格的部署测试。

### 12.5 系统侧不能夸大的内容

当前仓库能够支撑“系统实现了 SEM 图像导入、分割、统计、可视化和导出的一体化流程”的描述，但不应夸大为工业级全自动分析系统。模型推理依赖本地权重路径和运行环境，若权重文件缺失或路径配置不正确，深度学习分割不能直接运行。MatSAM、SAM LoRA、ResNeXt50 等模型槽位可以作为系统预留扩展能力描述，但除非有对应权重、推理代码和结果，否则不能写成已经完成完整对比实验的模型。

系统中的 OCR 标定、后处理预览和 Word 报告导出可以作为功能模块说明，但其准确性仍依赖输入图像质量、比例尺识别结果和人工确认。若论文中讨论真实材料学统计精度，需要补充人工标定或抽样核验记录。
