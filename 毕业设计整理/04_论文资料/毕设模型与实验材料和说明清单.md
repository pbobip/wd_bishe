## 二、训练流程真实情况

训练入口为：

```bash
python -m experiments.mbu_netpp.train --config experiments/mbu_netpp/configs/opt_real53_boundary_sampling.yaml --run-all-folds
```

主配置 `experiments/mbu_netpp/configs/opt_real53_boundary_sampling.yaml` 的真实参数为：`patch_size=256`，`overlap=0.25`，`samples_per_epoch=192`，`batch_size=6`，`epochs=80`，`learning_rate=0.0001`，`weight_decay=0.0001`，`optimizer=AdamW`，`scheduler=cosine`，`amp=true`，`device=cuda`，`threshold=0.5`，`boundary_tolerance=2`。

数据加载流程是：先由 `preparation.py` 将 LabelMe JSON 转换为二值 mask、edge mask 和 prepared manifest；训练时 `dataset.py` 读取 `experiments/mbu_netpp/workdir/prepared_real53_cv5/manifests/dataset.json` 和 `folds_5_seed42_holdout10.json`。训练集、验证集、测试集按原图 stem 划分，再从训练图像中在线随机裁剪 patch，因此当前主流程不是先离线增强成一批固定图像。

在线数据增强真实配置为：水平翻转 `p=0.5`、垂直翻转 `p=0.5`、随机 90 度旋转 `p=0.75`、仿射缩放 `p=0.3`、亮度对比度扰动 `p=0.25`、高斯噪声 `p=0.2`。训练阶段每 fold 看到 `192 × 80 = 15360` 个动态训练 patch，5 折共 `76800` 个动态 patch；3 个随机种子重复实验共 `230400` 个动态 patch。以上数量来自 `results/paper_supplement_all/augmentation_volume/augmentation_volume_report.json`。

模型保存和日志记录真实存在：每个 fold 保存 `history.json` 和 `summary.json`，实验级保存 `crossval_summary.json`，最终主模型保存到 `results/opt_real53_suite/experiments/opt_real53_boundary_sampling/best.pt`。Loss/指标曲线图真实存在，例如 `results/opt_real53_suite/experiments/opt_real53_boundary_sampling/plots/fold_0_training_curves.png` 和 `crossval_mean_curves.png`。

## 三、推理与分割流程真实情况

离线推理入口为 `experiments/mbu_netpp/infer.py`，参数包含 `--checkpoint`、`--input`、`--output-dir`、`--device`、`--threshold`。输入可为单张图或目录；输出目录包含 `masks`、`overlays`、`stats`、`summary.csv`、`summary.xlsx`。推理逻辑会读取灰度图，按配置做 CLAHE/minmax 等预处理，使用滑窗推理，sigmoid 后按阈值 0.5 得到二值 mask，并生成 overlay。

测试集评估入口为 `experiments/mbu_netpp/holdout_eval.py`。该脚本读取 `stage="test"` 的 holdout 样本，输出：

- `results/opt_real53_suite/experiments/opt_real53_boundary_sampling/holdout_eval/predictions`
- `results/opt_real53_suite/experiments/opt_real53_boundary_sampling/holdout_eval/overlays`
- `results/opt_real53_suite/experiments/opt_real53_boundary_sampling/holdout_eval/stats`
- `results/opt_real53_suite/experiments/opt_real53_boundary_sampling/holdout_eval/per_image_metrics.csv`
- `results/opt_real53_suite/experiments/opt_real53_boundary_sampling/holdout_eval/summary.json`

系统推理流程位于 `backend/app/services/pipeline.py` 和 `backend/app/services/runners/dl_infer.py`。前端选择 MBU-Net++ 后，后端通过 `ModelRunnerService` 调用独立 Python 进程执行 `dl_infer.py`，生成 mask/overlay，再由 `statistics_service.summarize` 生成统计指标。当前系统默认 runner 在 `backend/app/db/init_db.py` 中写死了本机 Python 路径和默认权重路径，换机器需要修改。

100 张未标注/待修正图片的推理输出真实存在：`results/real49_final_infer_100/infer_all_100`，包含 100 个 mask、100 个 overlay、100 个 stats JSON，以及 `summary.csv/xlsx`。当前 `dataset/full_png_cropped_xlsx/images` 中也有 100 个 PNG 和 100 个 JSON，但这些 JSON 属于已加入的 LabelMe 草稿/预测标注材料，不能直接当作人工真值标签使用，除非你已人工确认。

## 六、评价指标与 baseline 情况

深度学习指标真实存在于 `experiments/mbu_netpp/metrics.py`，包括 Dice、IoU、Precision、Recall、VF Error 和 Boundary F1。其中 VF Error 是预测前景面积占比与真值前景面积占比的绝对误差，适合 γ′ 相体积分数/面积分数统计误差分析；Boundary F1 用于反映边界对齐质量，适合说明 SEM 微观组织边界锯齿、细小通道等问题。

主模型 `opt_real53_boundary_sampling` 在 10 张 holdout 上的结果来自 `results/opt_real53_suite/experiments/opt_real53_boundary_sampling/holdout_eval/summary.json`：Dice `0.964984`，IoU `0.932976`，Precision `0.954685`，Recall `0.976702`，VF Error `0.024045`，Boundary F1 `0.929954`。

3 随机种子重复实验来自 `results/paper_supplement_all/seed_repeats/seed_repeat_summary/seed_repeat_summary.csv`。holdout Dice 分别为 `0.962391`、`0.966674`、`0.963097`，均值约 `0.964054`，标准差约 `0.002296`；holdout Boundary F1 均值约 `0.921499`，标准差约 `0.006112`；holdout VF Error 均值约 `0.025819`，标准差约 `0.003383`。这可以支撑模型结果稳定，但不能证明跨设备、跨来源完全泛化。

第一轮模型对比结果来自 `results/real53_full_experiments/suite_summary/suite_summary.csv`。其中 DeepLabV3+ ImageNet holdout Dice 为 `0.959546`，DeepLabV3+ MicroNet 为 `0.959167`，U-Net++ no pretrain 为 `0.963412`，U-Net++ MicroNet 为 `0.960833`，MBU Edge+Deep 为 `0.964216`。这些结果说明深度模型整体显著优于传统阈值路线，边界/深监督模块对 Boundary F1 有提升，但 MicroNet 在 DeepLabV3+ 上未表现出稳定优势，不能简单写成“预训练必然提升”。

## 七、模型和实验的可复现性判断

结论：基本可以复现，但需要补充若干文件和路径调整。

当前仓库具备训练、推理、评估、传统 baseline、曲线绘制、效率统计和结果打包脚本；本机也存在主数据 prepared、主权重、主要结果文件和 100 张推理输出。因此在这台机器上，沿现有路径大体可复查结果。

但不能说“任何环境直接复现”。主要原因如下：

- 主配置 `experiments/mbu_netpp/configs/opt_real53_boundary_sampling.yaml` 中原始图像路径写死为 `C:\Users\pyd111\Desktop\analysis_same_teacher_nocap_095\1`，不在仓库内。
- `backend/app/db/init_db.py` 写死了多个本机 Anaconda Python 路径，例如 `C:\Users\pyd111\anaconda3\python.exe`。
- 训练依赖只在 `experiments/mbu_netpp/requirements.txt` 中列出包名，未锁定版本；后端依赖在 `backend/requirements.txt`，前端依赖在 `frontend/package.json`，深度学习和系统运行环境需要分别配置。

## 八、论文中可以直接使用的描述

本研究面向扫描电镜图像中的镍基单晶微观组织开展 γ′ 相分割与统计分析。根据当前代码实现，任务被建模为二值语义分割问题：输入为灰度 SEM 图像，输出为 γ′ 相二值掩膜，其中前景像素表示 `gamma_prime`，背景像素表示 γ 基体或非 γ′ 区域。分割结果进一步用于计算 γ′ 相面积占比、颗粒数量、面积、等效直径、周长、长短轴、Feret 直径、圆度、长宽比和通道宽度等统计指标。

本文主模型可表述为基于 U-Net++ 的 MBU-Net++ 分割模型。该模型使用 `se_resnext50_32x4d` 作为编码器，并引入 MicroNet 显微图像预训练权重；在 U-Net++ 解码结构基础上，增加边界辅助监督和深监督输出，以增强 γ/γ′ 相界面区域和多尺度特征的学习能力。训练损失由 Dice+BCE 主分割损失、边界损失和深监督损失构成。当前主实验配置未启用 VF loss，因此不能写成最终主模型使用了体积分数损失。

数据方面可以写：主监督数据集包含 53 张 SEM 图像及其 LabelMe 多边形标注，所有标注统一为 `gamma_prime` 类别。程序审计显示 53 张图像均有对应 JSON，合计 4268 个 polygon，未发现缺失配对或无效 polygon。实验采用固定 10 张 holdout 测试集，其余 43 张用于 5 折交叉验证；训练阶段按原图划分后进行 256×256 patch 随机裁剪和在线数据增强，避免同一原图 patch 同时进入训练和验证/测试。

实验方面可以写：传统路线包括 Otsu 阈值、CLAHE+Otsu、自适应阈值和边缘检测路线；深度模型对比包括 DeepLabV3+、U-Net++、MBU-Net++ 及边界/深监督/VF/采样策略变体。最终主模型在 10 张 holdout 测试图上取得 Dice `0.964984`、IoU `0.932976`、Boundary F1 `0.929954`、VF Error `0.024045`。RTX 5090 上效率测试显示模型参数量为 `50,986,709`，256×256 patch 约 `115.37 GFLOPs`，holdout 平均推理时间约 `0.115 s/图`。

系统实现方面可以写：系统包含图像导入、SEM 底部区域裁剪、预处理、传统分割、深度学习分割、mask/overlay 可视化、统计分析、图表生成和结果导出模块。导出文件包括图像级指标、颗粒级统计表、批量汇总表、配置快照、分割 mask/overlay 压缩包和实验报告。
