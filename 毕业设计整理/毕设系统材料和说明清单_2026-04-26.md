## 1. 系统概述

**项目名称**：SEM 分割统计平台（SEM Microstructure Segmentation & Statistical Analysis Platform）

**核心定位**：一个面向扫描电镜（SEM）图像的全流程 Web 分析系统，提供图像预处理、分割、统计分析和结果导出的一体化解决方案。用户通过浏览器操作，后端自动完成从原始 SEM 图像到结构化统计报告的全部分析步骤。

**核心技术栈**：

| 层级 | 技术选型 | 用途 |
|------|---------|------|
| 后端框架 | FastAPI + Uvicorn | RESTful API 服务 |
| 数据库 | SQLite + SQLAlchemy 2.0 | 任务元数据、指标、导出记录持久化 |
| 图像处理 | OpenCV、scikit-image、NumPy、Matplotlib | 传统分割、去噪、增强、可视化 |
| OCR | RapidOCR (ONNX Runtime) | SEM 图像底部标尺和参数的自动识别 |
| 深度学习推理 | PyTorch（独立进程）、SAM/MBU-Net++ | 高精度图像分割 |
| Word 报告 | python-docx | 自动化生成实验报告文档 |
| 前端框架 | Vue 3 + Vite + TypeScript | SPA 用户界面 |
| UI 组件库 | Element Plus | 表单、卡片、对话框等组件 |
| 状态管理 | Pinia | 前端任务状态和配置管理 |
| 可视化图表 | ECharts | 面积分布直方图、体积分数柱状图 |
| 打包工具 | JSZip | 浏览器端多文件 ZIP 打包下载 |
| 部署 | Docker + Docker Compose + Nginx | 生产容器化部署 |

## 4. 后端服务层

### 4.1 流水线编排（pipeline.py）

核心引擎为 `PipelineService.execute(db, run_id)`，按以下 6 个步骤顺序执行：

```
步骤 1: input（整理输入图像）
  └── 对每张图像：ROI 检测 → 比例尺识别 → OCR 标定信息提取 → 分析区域裁剪 → 保存 layout

步骤 2: preprocess（图像预处理）
  └── 对每张图像（按配置）：
      - 背景校正（高斯滤波）
      - 主去噪（None / 小波 / 高斯 / 中值 / 双边 / 均值）
      - 主增强（None / CLAHE / Gamma / 直方图均衡化）
      - 可选锐化（Unsharp Mask）

步骤 3: traditional（传统分割）[分割模式 ≠ 'dl' 时执行]
  └── 对每张图像（按配置）：
      - 阈值（Otsu / 全局阈值 / 固定阈值）
      - 自适应阈值（Gaussian / Mean）
      - 边缘（Canny / Sobel / Laplacian）
      - 聚类（K-Means）
      前景目标可选：暗相（γ′ 颗粒）或亮相（通道区域）

步骤 4: dl（深度学习分割）[分割模式 ≠ 'traditional' 时执行]
  └── 对每张图像：
      - 调用 model_runner_service，生成子进程 JSON 配置
      - 子进程加载模型（MBU-Net++ / MatSAM / Custom）
      - 推理后输出 mask.png + overlay.png
      - 进度写入 progress JSON 文件（pipeline 通过轮询读取）

步骤 5: stats（统计摘要与图表）
  └── 对每张图像（按确认的分割模式）：
      - Volume Fraction（VF）= 前景像素数 / 总像素数
      - 颗粒数量统计
      - 面积/尺寸分布直方图（PNG，Matplotlib）
      - 通道宽度（Distance Transform + 骨架化）
      - Lantuéjoul 边界校正
      - 95% 置信区间（t 分布）

步骤 6: export（结果导出）
  └── 生成文件：
      - xlsx（图像级主统计表）
      - batch_xlsx（批次汇总，含 95% CI）
      - particles_xlsx（颗粒明细，最多 500 条/表页）
      - config.json（任务配置快照）
      - segmentation_overlays.zip（分割叠加图 ZIP）
      - segmentation_masks.zip（分割掩膜 ZIP）
      - experiment_report.docx（Word 实验报告）
      - run_N_bundle.zip（全量打包）

```

### 4.2 任务调度（task_manager.py）

- 使用 `concurrent.futures.ThreadPoolExecutor(max_workers=2)` 调度任务
- 每个任务在独立线程中调用 `pipeline_service.execute()`
- 任务启动时状态从 `draft` → `queued`，完成后 `running` → `completed`/`failed`
- 支持任务取消（待扩展）

### 4.3 模型运行器（model_runner.py + dl_infer.py）

**架构**：模型推理在独立子进程中运行，通过 JSON 文件交换数据（避免 CUDA 内存冲突）。

**流程**：
1. `model_runner_service` 写入 `config.json`（含模型类型、图像列表、权重路径、输出目录）
2. 启动子进程：`python dl_infer.py config.json`
3. 子进程加载模型，执行推理，输出 `mask.png` + `overlay.png`
4. 子进程通过 `progress.json` 实时报告进度（`processed/total`）
5. 推理完成后写入 `manifest.json`，主进程解析结果

**支持的模型**：

| 槽位 | 模型 | 环境 | 功能 |
|------|------|------|------|
| `mbu_netpp` | MBU-Net++ | base (conda) | 高精度分割（SE-ResNeXt50 + U-Net++ + Edge Head + Deep Supervision） |
| `matsam` | MatSAM (SAM automatic masks) | sam_gpu | 全自动分割，无需提示点 |
| `sam_lora` | SAM LoRA | sam_gpu | 预留 |
| `resnext50` | ResNeXt50 | torch311 | 预留 |
| `custom` | 自定义阈值分割 | aavt_gpu | 快速占位分割（阈值可配置） |

**MBU-Net++ 详情**（`dl_infer.py:run_mbu_netpp`）：
- 加载 PyTorch checkpoint（`best.pt`），权重路径由 `ModelRunner` 表管理
- 输入尺寸 256×256，滑动窗口重叠率 0.25，支持 CUDA/CPU 自动选择
- 自动裁剪 SEM 底部 footer 区域（基于 Canny 边缘检测）
- 内置预处理（按训练时配置）：归一化 + 灰度化

### 4.4 传统分割（algorithms/traditional.py）

支持 4 条路线，每条路线均可选"暗相"（γ′ 颗粒）或"亮相"（通道区域）前景目标：

| 路线 | 核心算法 | 关键参数 |
|------|---------|---------|
| `threshold` | Otsu / 全局阈值 / 固定阈值二值化 | threshold_mode, global_threshold, fixed_threshold |
| `adaptive` | 自适应阈值 | adaptive_method, adaptive_block_size, adaptive_c |
| `edge` | 边缘检测（Canny/Sobel/Laplacian）+ 形态学闭合 | edge_operator, edge_threshold1/2, edge_dilate_iterations, edge_blur_kernel |
| `clustering` | K-Means 聚类 | kmeans_clusters, kmeans_attempts, cluster_target |

### 4.5 OCR 标定（sem_footer_ocr.py + sem_roi.py）

- **ROI 检测**：`sem_roi_service.extract(image)` 裁剪底部 footer 区域
- **比例尺定位**：`detect_scale_bar(footer_image)` 通过灰度投影定位比例尺横线
- **OCR 识别**：`sem_footer_ocr_service.analyze(footer_image)` 使用 RapidOCR 识别：
  - 比例尺数值（μm）、视场（μm）、放大倍率、工作距离、扫描模式、真空模式、日期时间
- **自动标定**：若比例尺数值和像素长度均识别成功，自动计算 `um_per_px` 建议值
- **降级策略**：比例尺识别失败时仅提示手动填写；OCR 不可用时仍可手动输入

### 4.6 图像预处理（preprocess.py）

```
背景校正 ──► 主去噪 ──► 主增强 ──► 可选锐化
```

| 阶段 | 选项 |
|------|------|
| 背景校正 | None / 高斯滤波（radius 可调） |
| 主去噪 | None / 小波（wavelet_strength）/ 高斯（gaussian_kernel）/ 中值（median_kernel）/ 双边（diameter, sigma_color, sigma_space）/ 均值（mean_kernel） |
| 主增强 | None / CLAHE（clip_limit, tile_size）/ Gamma（γ 值）/ 直方图均衡化 |
| 可选锐化 | Unsharp Mask（radius, amount） |

### 4.7 统计分析（statistics.py）

核心函数 `statistics_service.summarize(mask, um_per_px, objects, config)` 输出：

| 指标 | 说明 |
|------|------|
| `volume_fraction` | 前景像素占比（0~1），乘 100 即为 VF% |
| `particle_count` | 分割出的连通域数量（去除边缘接触颗粒后） |
| `mean_area` | 平均面积（含 Lantuéjoul 边界校正） |
| `mean_size` | 平均等效直径（等效圆直径） |
| `channel_width_x/y` | 通道宽度：Distance Transform + 骨架化统计 |
| `area_histogram` | 面积分布直方图数据 |
| `objects` | 各颗粒详细特征（area, perimeter, major/minor, feret, circularity, solidity, aspect_ratio...） |
| `ci95_*` | 各指标的 95% 置信区间（t 分布，样本数 ≤ 30 时使用临界值表） |

### 4.8 后处理预览（postprocess_preview.py）

采用"预览 → 确认"双阶段模式，避免误操作：

| 后处理操作 | 说明 |
|-----------|------|
| Fill Holes | 填充封闭孔洞 |
| Watershed | 分水岭分割（分离粘连颗粒） |
| Smoothing | 形态学开闭运算平滑边缘 |
| Shape Filter | 按面积/圆度过滤小噪声或大杂质 |
| Remove Border | 去除接触图像边界的颗粒（不计入统计） |

**流程**：创建预览 → 用户确认 → 全量重算统计 → 覆盖原分割结果和统计指标

### 4.9 Word 报告生成（report_service.py）

任务完成后自动生成 `.docx` 实验报告，包含：

| 章节 | 内容 |
|------|------|
| 封面 | 任务名称、创建时间、分割模式、图像数量 |
| 标定信息 | um/px 值、OCR 识别到的 SEM 参数 |
| 图像结果 | 原图 + 分割掩膜 + 叠加效果图（按图像分节展示） |
| 统计汇总 | VF、颗粒数、平均面积、平均尺寸、通道宽度（95% CI） |
| 统计图表 | 面积/尺寸分布直方图、VF 柱状图（嵌入 PNG） |
| 颗粒明细 | 各颗粒详细特征表（最多 500 行/表页） |
| 配置快照 | 完整 JSON 配置记录 |

报告通过 `python-docx` 生成，支持中英文混排（微软雅黑 + Times New Roman），蓝色主题色表格，自动分页。

## 8. 关键业务流程

### 8.1 任务执行完整流程

```
用户操作（前端）
  │
  ├─► 选择图像文件（支持 .png/.jpg/.jpeg/.tif/.tiff，多张或文件夹）
  ├─► 设置分割模式（traditional / dl / both）
  ├─► 配置分割参数（传统阈值参数 或 DL 模型槽位和权重路径）
  ├─► 配置预处理（可选：去噪 + 增强 + 锐化）
  ├─► 逐张标定（OCR 自动识别比例尺 或 手动输入 um/px）
  └─► 点击"开始处理"
           │
           ▼
  API: POST /runs  → 创建任务（draft）
  API: POST /runs/{id}/images  → 上传图像
  API: PUT  /runs/{id}  → 更新标定值
  API: POST /runs/{id}/execute  → 提交执行
           │
           ▼
  后端 ThreadPoolExecutor 接收任务
           │
           ▼
  ┌─────────────────────────────────────────────────┐
  │ PipelineService.execute(run_id)                 │
  │                                                 │
  │  ① input: ROI检测 → 比例尺定位 → OCR解析       │
  │  ② preprocess: 去噪 + 增强（可选）             │
  │  ③ traditional/dl: 分割（子进程独立推理）       │
  │  ④ stats: VF + 计数 + 分布 + DT通道宽度       │
  │  ⑤ export: XLSX + DOCX + ZIP bundle            │
  └─────────────────────────────────────────────────┘
           │
           ▼
  前端轮询 GET /runs/{id} 获取 progress 和 status
           │
           ▼
  status = "completed"
           │
           ▼
  前端跳转至结果页面 → 展示分割叠加图 → 用户可进行后处理
           │
           ▼
  用户点击导出 → StatsSidebar 打包下载（ZIP / 单独下载 Word）
```

### 8.2 后处理流程

```
用户查看分割结果
         │
         ▼
点击"后处理" → 选择操作（Fill Holes / Watershed / Smoothing / Shape Filter / Remove Border）
         │
         ▼
API: POST /runs/{id}/postprocess/preview
         │
         ▼
后端生成预览图（前/后对比）+ 预览统计变化
         │
         ▼
前端展示预览对话框
         │
         ▼
用户确认 → API: POST /runs/{id}/postprocess/confirm
         │
         ▼
后端重算统计 → 覆盖原分割结果 → 任务重新导出
```

### 8.3 Word 报告生成流程

```
Pipeline 导出步骤（pipeline.py → exporter.py）
         │
         ▼
exporter.export_run() 调用 generate_docx_report()
         │
         ▼
report_service.py 生成文档：
  ① 封面（任务名/模式/时间）
  ② 标定信息表（OCR 参数 + um/px）
  ③ 图像结果（原图 + 掩膜 + 叠加，按图像分节）
  ④ 统计汇总表（VF / 计数 / 均值 / CI95）
  ⑤ 图表嵌入（面积直方图 + VF 柱状图 PNG）
  ⑥ 颗粒明细表（截断至 500 行/表页）
  ⑦ 配置快照（完整 JSON）
         │
         ▼
保存至 storage/runs/{id}/exports/experiment_report.docx
写入 ExportRecord(kind="docx_report") 至数据库
追加至 run_{id}_bundle.zip
         │
         ▼
前端 StatsSidebar 显示"实验报告 Word"选项
用户可勾选下载或打包
```

## 9. 技术亮点与创新点

### 9.1 系统架构层面

1. **子进程隔离推理**：深度学习推理在独立 Python 子进程中运行，通过 JSON 文件通信，避免 CUDA 上下文冲突，同时支持多张图像并行推理。
2. **预览确认双阶段模式**：后处理操作先预览效果，用户确认后再批量重算，避免误操作导致分割结果不可逆。
3. **全链路异步任务管理**：FastAPI + ThreadPoolExecutor + 轮询进度，前端实时感知任务执行状态。
4. **自动化报告生成**：任务完成后自动生成 Word 格式实验报告，包含原图对比、统计图表和配置快照，无需人工整理。

### 9.2 图像处理层面

5. **SEM 图像自动标定**：结合 ROI 检测 + 比例尺定位 + RapidOCR，自动从 SEM 图像底部提取标定信息，减少人工操作。
6. **可配置预处理流水线**：去噪 + 增强 + 锐化可任意组合，支持 CLAHE、Gamma、小波等多种算法，适应不同 SEM 图像质量。
7. **Lantuéjoul 边界校正**：统计阶段对接触边界的颗粒进行边界效应校正，提高 VF 估算精度。
8. **Distance Transform 通道宽度**：通过距离变换骨架化自动计算相通道宽度，为材料微观结构分析提供量化指标。
9. **95% 置信区间**：对批次汇总指标计算 t 分布置信区间，量化测量不确定度。

## 10. 已知限制与改进方向

| 问题 | 当前状态 | 可能的改进方向 |
|------|---------|--------------|
| Uvicorn `--reload` 不检测新文件 | 需重启后端进程才能加载新 Python 模块 | 使用 `watchdog` 监控或手动重启 |
| 深度学习模型推理在 CPU 上速度慢 | 支持 CUDA 加速（需 GPU 环境） | 优化批处理推理、INT8 量化 |
| Word 报告生成失败不影响导出 | 当前实现静默忽略异常 | 添加失败提示，记录日志 |
| 后处理操作仅作用于传统分割 | DL 分割结果的后处理暂不支持 | 扩展后处理服务支持 DL 模式 |
| 任务无法中断/取消 | 当前仅支持任务执行完成 | 接入 ThreadPoolExecutor.cancel() |
